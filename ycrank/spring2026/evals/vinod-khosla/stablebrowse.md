# StableBrowse -- Vinod Khosla Evaluation

The first thing I notice about StableBrowse is the competitive landscape: over $180 million has already poured into browser infrastructure for AI agents. Browserbase alone has raised $67.5 million, has a thousand customers, and is doing $4.4 million in revenue. Kernel has Accel backing. Browser Use has 37.9k GitHub stars. When I see that level of consensus capital flowing into a category, my framework tells me the return profile is wrong for how I invest. This is not a contrarian bet — this is a category where every venture investor in San Francisco agrees the market is real. The experts agree. The VCs agree. And when everyone agrees, the probability-adjusted return compresses to something that doesn't interest me. I'm not looking for the popular bet. I'm looking for the bet that requires an apology letter to my LPs.

Let me be clear about the consequence question. If StableBrowse fully succeeds — dominates the semantic browser engine category — what changes for humanity? Developers building AI agents save money on infrastructure. Agents browse the web faster and cheaper. This is real, but it's a cost optimization for builders, not a transformation of how billions of people live. Compare this to Commonwealth Fusion Systems: if fusion works, energy becomes limitless. Compare it to Impossible Foods: if heme synthesis works, you feed the planet without animal agriculture. If StableBrowse works, browser sessions cost 10x less per instance. That's a good business. It's not a changed world. The consequence ceiling here is "successful infrastructure company" — and I don't invest my own money for that outcome. I either want to lose it all or make 50 times.

The strongest argument for this company is the technical architecture. I'll give credit where it's deserved: stripping the rendering pipeline from a browser engine while retaining JavaScript execution and network handling is genuinely non-linear. Every competitor — Browserbase, Kernel, Airtop — is managing fleets of standard Chromium instances. They're optimizing around the existing architecture. StableBrowse is rebuilding the engine itself. If the claimed metrics hold — 85MB RAM, 95% token reduction, sub-80ms cold start — that's not a 2x improvement, it's a step-function change. This is the one dimension where my interest gets activated. The architectural divergence means incumbents would have to cannibalize their own core infrastructure to respond, which is precisely the kind of structural advantage I look for. This is the Rocket Lab pattern: build the engine differently, don't just broker cheaper access to existing rockets.

But here's where I stall. The bull case requires me to believe that four founders — none with publicly verifiable experience in browser engineering, systems infrastructure, or anything close to the domain — can build a production-grade custom browser engine that handles the full insanity of the modern web. SPAs, Shadow DOM, iframes, WebSockets, the endless chaos of web compatibility. Chromium has thousands of engineers and decades of accumulated edge-case handling. These founders are recent graduates. Somansh Shah finished Purdue in 2025. No one on the team has a visible open-source project, a public GitHub presence in browser internals, or a track record building infrastructure at scale. I love outsider founders — I backed a 19-year-old dropout building weapons systems at Mach Industries. But that founder had already demonstrated extraordinary learning velocity visible to anyone paying attention. Here, I have no signal. No evidence of the rate of learning. No evidence of adaptation. Four people with a website full of impressive performance claims and no public benchmarks, no customers, no GitHub repos, and no way for me to assess whether they are first-principles thinkers or repeating a pitch deck they wrote for YC. When the technical ambition is this high, the founder signal needs to be proportionally strong. It's not.

If I'm wrong about this team — if one of these founders turns out to be an extraordinary systems engineer who's been building browser internals in private for two years — this could be a compelling option bet. The technical approach is genuinely differentiated. The market is clearly forming. The cost advantage, if real, creates switching costs. The scenario where StableBrowse becomes the default infrastructure layer for the agentic web is not absurd. But that scenario requires extraordinary execution against well-funded incumbents, in a consensus category, by an unproven team. The option price isn't the problem — I'll gladly lose a million dollars. The problem is that the consequence magnitude doesn't justify the attention. Even in the best case, this is a successful infrastructure company, not a company that makes the world fundamentally different. I pass.
