# Replicas -- Elad Gil Evaluation

The first thing I see here is a company building on top of an infrastructure layer it doesn't control, in a market where the infrastructure owners have direct incentive and capability to subsume the product. OpenAI's Codex already runs in a cloud sandbox. Anthropic is shipping Claude Code Channels. When I look at a company's positioning, the question that matters most is: are you the platform, or are you the app? Replicas is the app. It wraps Claude Code and Codex in a VM, plugs into Slack and Linear, and dispatches tasks. The core technical operation -- spin up a VM, clone a repo, run a CLI command -- is operationally straightforward. This is not Stripe building the payments API layer that ten thousand companies integrated into their billing stacks. This is a convenience wrapper that lives or dies based on whether the upstream providers choose to build native cloud execution with workflow hooks. One of my active red flags is "startup is building an application on a contested infrastructure layer." That's exactly what I'm looking at.

The market timing question here is telling, because the structural shift is real but entirely consensus. CLI-based coding agents that run headlessly emerged in mid-2025, and yes, that's a genuine capability threshold that created new demand for orchestration and cloud execution. But look at the capital already deployed: Devin has raised $900M+ at a $10B+ valuation, Factory raised $50M at $300M, Augment Code has $252M behind it. This is not a nonobvious market -- it's one of the most crowded, well-funded categories in all of tech right now. When I invested in Coinbase, nobody else wanted to touch crypto infrastructure. When I wrote the seed check for Perplexity, AI search was considered a failed category. The alpha in those investments came from being right when the market was wrong. Here, everyone agrees this market is huge. The question isn't timing -- it's whether a solo founder with $500K can outcompete companies with 100-1000x more capital in a race everyone is already running.

The bull case deserves examination. There's a coherent Terraform analogy: just as Terraform became the neutral infrastructure-as-code layer across competing cloud providers, Replicas could become the neutral agent orchestration layer. Anthropic and OpenAI have limited incentive to build multi-agent dispatch that features each other's agents, which creates a structural gap for a neutral platform. If engineering teams standardize on dispatching agent work through Slack and Linear rather than individual CLIs, the workflow integration layer becomes the sticky surface. The 30%+ PR completion rate suggests real product utility. And being agent-agnostic means Replicas improves passively as any underlying agent improves. If I squint, there's a version of this that captures the "boring operational infrastructure" pattern I've invested in repeatedly -- the unglamorous orchestration layer that every team needs but nobody's prestige is attached to building. That version would require the agent landscape to fragment further (five or six viable agents, not two), and for none of the agent providers to build native multi-agent dispatch. Both are plausible but neither is certain, and the combination starts to look like compounding conditional probabilities.

The single-miracle test fails here in a specific way. The company needs the behavioral shift to asynchronous agent dispatch (plausible, and underway). But it also needs the major agent providers to NOT build native cloud execution with workflow integrations -- a negative miracle, hoping that incumbents with enormous resources and clear incentive don't act. And it needs to outcompete or survive alongside Devin, Factory, and Augment Code with dramatically less capital and a one-person team. That's at least two independent miracles, which is where my "compounding small odds" concern kicks in. When I overrode this heuristic for Anduril, it was because the market structure opening was generational and bipartisan defense reform created a window that no one else was positioned to exploit. Here, the market opening is well-understood and heavily contested.

Connor Loi has the right instincts -- building developer infrastructure, shipping at Helicone, choosing YC. His 8 months on the Helicone core team building the AI Gateway is genuinely relevant experience for this problem. But he's a solo founder, a current or recent student, with no prior founding experience and no go-to-market co-founder. The company isn't hiring. When I think about what it takes to compete against Devin's 100+ person team or Factory's enterprise sales motion, one person running product, engineering, sales, and support simultaneously isn't a scaling architecture -- it's a bottleneck. I've seen strong markets compensate for execution gaps, but the market here isn't structurally favorable enough to carry that weight.
