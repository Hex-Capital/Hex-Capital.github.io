# Stage -- Naval Ravikant Evaluation

CodeRabbit has $15M ARR and $88M in funding. Graphite was acquired by Cursor for $290M. Greptile and Qodo have raised $165M between them. Stage enters this market with two people and a UX concept — "structured chapters" for code diffs. The combined competitive war chest exceeds $334M. I don't avoid competitive markets on principle — I invested in Uber when dozens of car services existed. But Uber had Travis Kalanick's specific knowledge about marketplace dynamics and a permissionless leverage architecture that incumbents structurally couldn't replicate. Stage's "chapters" innovation is a presentation-layer pattern. Any front-end team at CodeRabbit or GitHub could ship this in a quarter. When your moat is a UI concept rather than deep technology or accumulated specific knowledge, you're building on sand.

Charles Pan and Dean Stratakos are both Stanford CS graduates who worked together at Five Rings. Stratakos led the firm's company-wide AI initiative and built an internal coding agent — relevant to Stage's thesis that AI-generated code creates a comprehension gap. Pan shipped as an early engineer at Yuzu Health. These are competent builders with relevant experience. But relevant is not the same as non-replicable. Thousands of engineers at quantitative trading firms experience code review pain daily. Thousands more have built internal AI coding tools. If I ask my foundational question — "If these founders disappeared, would Stage's core insight survive?" — the answer is clearly yes. Any thoughtful engineer who has reviewed AI-generated PRs could arrive at the chapters concept. Contrast Stratakos's resume with Aravind Srinivas spending years at Google Brain and OpenAI building the exact retrieval systems that power Perplexity — that's knowledge so deep a competitor needs years to accumulate it. Stage's founders have breadth, not depth, in this specific domain.

Stage's leverage architecture looks like standard SaaS on paper — code that scales per seat without proportional headcount. But the leverage audit reveals a dependency problem. Stage sits on top of GitHub or GitLab's platform. The structural barrier protecting Stage from GitHub adding a native "chapters view" is near-zero — unlike Google's ad revenue model that prevents it from cannibalizing search with AI answers, GitHub faces no cannibalization risk from improving its diff viewer. Adding an optional structured view is an additive feature, not a product rewrite. This means Stage's leverage is rented, not owned. When I backed Notion at the seed, Notion owned its entire surface area — blocks, databases, collaboration — and switching costs compounded with every team template created. Stage's switching cost is whatever time a team spent configuring chapter preferences. That's a weekend of engineering to replicate elsewhere.

The bull case is real and I want to steelman it honestly. Cursor acquired Graphite for $290M — proof that code review is a strategic acquisition category. Stage's two-person team is exactly the lean structure I've championed since 2012. Both founders are technical builders who left lucrative quant finance jobs, which shows at least some principal commitment. The timing catalyst is genuine: AI-generated code volume is growing faster than reviewer bandwidth, and the comprehension gap is a new problem that existing tools don't solve well. An advocate would argue Stage could become the "structured comprehension layer" that every AI coding tool needs, the way Notion became the knowledge layer every team needs. If Stage showed evidence of obsessive early-user love — developers who tried chapters and refused to go back to raw diffs — I would lean differently. If Stratakos had published research on code cognition, built open-source code comprehension tools before Stage, or demonstrated years of specific engagement with how developers read code rather than how they write it, that would shift the specific knowledge score meaningfully. One verifiable condition that flips me: 50+ engineering teams using Stage daily with retention above 80% at week four, demonstrating that the chapters paradigm creates genuine workflow lock-in rather than a marginal improvement.

Stage doesn't fit my access democratization pattern. Code review is already universally accessible — GitHub PRs are free. Stage makes review marginally better organized for teams already well-served by existing tools. Compare this to AngelList giving thousands of non-connected investors access to angel deals previously restricted to a few hundred insiders, or CoinList opening token launches to retail. Stage expands access to nothing new. It optimizes an existing workflow. Innovation without a moat becomes consumer surplus — value flowing to users and eventually to the platform that absorbs the feature.

The dimension that locks my verdict is Specific Knowledge Productization. Stage's founders are smart generalist engineers who identified a market opportunity — AI code creates a review comprehension gap. That's an observation, not specific knowledge. The chapters concept is replicable by any competitor with design resources, which is why CodeRabbit, Greptile, and Qodo have already shipped structured review features in 2025-2026. The second decisive dimension is Permissionless Leverage Architecture — Stage's leverage is platform-dependent and non-compounding, with no data flywheel or network effect evident in the dossier. Two talented builders with a UX insight entering a $334M competitive battlefield is courage, but courage without specific knowledge is just a coin flip. I pass.
