# Memory Store -- Paul Graham Evaluation

Memory Store is what happens when capable shippers point themselves at a category everyone in the room already agrees is hot. Diwank Tomer and Ishita Jindal spent three years building Julep to $400k ARR and 6.6k GitHub stars, and now they've extracted the `src/memory-store` directory from that monorepo and turned it into a company in a category where Mem0 has already raised $24M and Letta has 13k stars. The founder quality is real. The idea sits in exactly the part of the Venn diagram I avoid.

On founder resourcefulness, the ladder runs from Airbnb (cereal boxes funding the company when no investor would touch it) at the top, through Stripe (Patrick Collison having already shipped Auctomatic before he was old enough to drink in the US) in the middle, down to the McKinsey-pair pitching B2B workflow automation at the bottom. Memory Store sits between Stripe and the middle rung — closer to Stripe than to the consultants. Diwank ran engineering at Jar and was CEO of Whitehead AI before Julep; together with Ishita he shipped an open-source agent platform that real developers actually used. That is not nothing. What I don't have is a cereal-box story — a moment where Memory Store specifically forced these founders into creative constraint. The Julep ARR is the artifact I'm scoring on, and it's a solid one.

On the dimension that hurts most — whether the idea repels before it attracts — the ladder runs from Stripe (payments and PCI compliance, a schlep so unglamorous most developers wouldn't touch it) through ClassDojo (K-12 procurement, which VCs hated for sales-cycle reasons), down to the consultants pitching B2B workflow automation that "everyone agreed was a great opportunity." Memory Store is sitting on the bottom rung. The dossier literally says it was "voted #1 most promising startup when their YC W26 batch kicked off." That's the tell. When I read about Mem0's $24M Series A, Letta, Zep, Supermemory, Cognee, LangMem, and Hindsight all shipping overlapping memory primitives, plus Anthropic having already shipped native memory in Claude in 2025, what I see is a category where the returns have already been competed away. The best startup ideas sound like bad ideas. "AI memory layer that syncs your tools" is the opposite — it sounds great to everyone, which is exactly when I get nervous.

On organic problem discovery, the ladder is Dropbox (Drew Houston forgot his USB drive) at the top, Replit (hackers building tools they themselves wanted) in the middle, and the McKinsey consultants who "identified an opportunity through 50 customer interviews" at the bottom. Memory Store's stated origin — "four months of talking to founders about how they use AI" — sits between Replit and the bottom. It's market research framing. The redeeming fact is that Memory Store is literally extracted from a directory inside Julep, which means at some point during three years of agent-platform building these founders hit the memory problem in their own work and built a primitive for it. That's organic. The "four months of interviews" framing is the part that worries me, because it's how founders talk when they discovered an idea by asking rather than by living it.

On founder learning rate, the rungs are Sam Altman at 19 running Loopt (a learning slope that justified championing him for YC president), Reddit's pivot from cell-phone food ordering to link aggregation in the middle, and the year-of-no-iteration founder at the bottom. Memory Store sits closer to Reddit than to Altman — a real pivot from Julep's full agent platform to a single extracted layer, in response to where actual users were getting value. Tom Blomfield as partner is a quiet positive; he's a founder-mode partner. On earnest builder identity, the ladder goes Watsi (the only board seat I've ever taken, zero financial return) at the top, Replit (hackers obsessed with making programming accessible) in the middle, and pure-status founders at the bottom. Memory Store is mid — these are clearly builders, the OSS Julep work proves it, but the "voted #1 most promising" line in the launch material is exactly the kind of fundraising-as-achievement framing I distrust.

My 36-month forecast: Memory Store gets acqui-hired in the $20–50M range by Notion, Mem0, or Anthropic itself — most likely Mem0, which has the capital and the developer footprint to want the SaaS/MCP packaging Memory Store builds. ARR before acquisition lands somewhere between $800K and $3M. That outcome is the natural product of the signature here: high founder execution, low idea differentiation, in a category with a funded leader and a platform owner (Anthropic) capable of absorbing the wedge. Good outcome for the founders, mediocre outcome for early checks.

The strongest bear-of-my-bear case: I should re-read this as a Stripe analog. Patrick Collison had already shipped Auctomatic when I wrote him the $15K check; Diwank and Ishita have already shipped Julep to real ARR. The "follow the founder" pattern is alive here. Maybe Memory Store is the second swing where the founders, having learned the agent-infrastructure market the hard way, pick the right wedge. The case stings — I do back repeat shippers. But the dispositive fact is that Stripe was tackling a schlep that repelled everyone (PCI compliance, bank integration), while Memory Store is tackling a category where eight funded teams are already competing on benchmark scores. Schlep blindness was Stripe's moat. Memory Store has the opposite — competitive density. The condition that flips me: Memory Store posts a public LongMemEval score above Supermemory's 85.4% AND signs a named team-tier customer (not an individual seat) at $50K+ ACV. Either alone is noise; both together would mean the team-shared MCP wedge is actually defensible and I'd write a check.
