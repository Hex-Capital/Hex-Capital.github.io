# primitive -- Sam Lessin Evaluation

Here's the thing that jumped out at me immediately: this company named itself after the exact concept I look for in investments, and then proceeded to not actually be one. "Primitive" — email as a developer primitive for AI agents. I love the vocabulary. But calling yourself a primitive doesn't make you a primitive. Email already exists. Webhook routing already exists. SendGrid and Mailgun have been doing programmatic email for over a decade. What Primitive is building is a better adapter between existing email infrastructure and the hot new thing — AI agents. That's not a primitive. That's a feature. A primitive creates a behavior that couldn't be described before it existed. Venmo created "Venmo me." What does Primitive create? "Route my agent's email to a webhook"? That's a configuration, not a category.

The mispricing problem here is fatal from my perspective. Agent infrastructure is the single hottest consensus category in venture right now. AgentMail — same space, one YC batch earlier — raised $6M from General Catalyst with Paul Graham and Dharmesh Shah as angels. Their user counts tripled in a week. Gartner is publishing projections about 40% enterprise agent adoption. Every sophisticated investor on the planet is looking at agent tooling and thinking "this is the next platform shift." When the market is this excited, the upside is already in the price. You're buying at the top of the consensus curve. I want to buy when everyone thinks I'm an idiot — that's where the asymmetry lives. Here, I'd be buying when everyone agrees this is a great category, which means I'm paying full price for the optionality. That's a sucker bet.

Now, the bull case — and I genuinely considered it. What if email becomes the default communication bus between autonomous agents, the way HTTP became the default for web services? And what if Primitive's webhook-first architecture is architecturally superior to AgentMail's mailbox model for developer adoption? The Twilio analogy is tempting: Twilio turned telephony into an API and built a $50B+ company. If Primitive could be "Twilio for agent communication," the scale is enormous. But the analogy breaks down on the moat. Twilio worked because telephony required carrier relationships, physical network infrastructure, phone number provisioning — real-world complexity that software alone couldn't replicate. Email infrastructure is commoditized. AWS SES plus a Lambda function gets you 80% of the way there. Google's Agent2Agent protocol, now a Linux Foundation project with 100+ partners, is building the actual protocol layer for agent interoperability — the thing I'd actually want to invest in. Primitive isn't the protocol. It's an app sitting on top of protocols controlled by others.

The founder has real credentials — Google, Meta, AWS, Microsoft, plus founding Actual AI which raised $3.2M and had 32 active pilots. That's a sane person, no question. But the idea isn't insane. Nobody hears "email infrastructure for AI agents" and thinks "that's fucking crazy." They think "yeah, obviously someone should build that." That's the problem. My best investments are when smart people tell me I'm wrong and the founder can calmly explain why they're right about something counterintuitive. Nothing about this pitch is counterintuitive. And the Actual AI situation nags at me — his LinkedIn still says "Founder & CTO @ Actual AI," which had a co-founder, $3.2M, and 32 pilots. What happened? Is Primitive a pivot? A second venture? The ambiguity around leaving a funded company with traction to start solo in an adjacent space with no traction is something I'd need answered before going further. Solo founder against AgentMail's three co-founders with a $6M war chest and 500+ customers — that's a real execution gap.

The structural durability question is where this really falls apart. Every time I look at a developer tool and ask "could an incumbent add this as a feature?" — and the answer is obviously yes — I pass. AWS, Google Cloud, Microsoft Azure all have email services. All have agent platforms. All have webhook infrastructure. Adding an "agent email" product line is a Tuesday afternoon feature sprint for any of these companies. The core technology — routing email to a webhook endpoint — is not defensible. There are no physical-world moats here, no protocol-level network effects, no regulatory licenses, no proprietary data from real-world operations. This is pure software doing something technically reproducible, which is exactly the category I've been saying publicly for years gets commoditized by AI and incumbents. "Software companies that had moats are learning they aren't moats anymore" — and this company doesn't even have moats yet.
