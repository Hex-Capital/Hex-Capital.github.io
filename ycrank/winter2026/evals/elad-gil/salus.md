# Salus -- Elad Gil Evaluation

The first thing I notice about Salus is the competitive landscape -- and it's not encouraging. Fiddler AI has $100M in total funding and has already shipped a Realtime Guardrails API that directly competes with Salus's core value proposition. Guardrails AI has an open-source framework with 5.9K GitHub stars providing a free baseline. Galileo AI was acquired by Alphabet for $68M+. When I map this market, I don't see a "too boring" or "too niche" category that other investors are overlooking -- I see a consensus-hot AI safety market where multiple well-funded teams are already converging on the same insertion point. That's the opposite of my best investment pattern. PagerDuty worked because nobody else wanted to build operations alerting. Salus is building in a space where everybody wants to build.

The structural moment is real but obvious. AI agents transitioning from text generators to action executors -- function calling, MCP protocol, tool use proliferating across every major framework -- creates a genuine new failure mode. When an agent processes a refund without verifying an order ID, that's a real-world cost, not a hallucination in a chat window. Gartner projecting 40% of enterprise applications embedding AI agents by end of 2026 validates the timing. But when Gartner is already projecting it, it's not a non-obvious market. The TAM slides showing $2.5B growing at 23.7% CAGR are exactly the kind of static market sizing I warn about -- they tell you this is a category, not that THIS company will win it. The more dangerous question is whether the LLM providers themselves -- OpenAI, Anthropic, Google -- will build native runtime validation into their SDKs. Salus argues their cross-platform positioning protects them, but that's a thin moat when each provider has massive incentive to make their own agents safer. If Anthropic ships native tool-call validation in their SDK, Salus's decorator wrapper becomes redundant for every Anthropic-based agent overnight.

The product-to-distribution trajectory concerns me. A Python SDK decorator that wraps tool calls is a useful integration point but a shallow one. Stripe's API wasn't just a payment method -- it was a developer relationship that naturally expanded into Atlas, Billing, and Treasury because every developer who processed payments also needed business formation, invoicing, and banking. What does Salus expand into? More types of validation? That's feature expansion, not distribution leverage. The evidence cache mechanism could potentially evolve into an agent memory or reasoning audit layer, but the founders haven't articulated that multi-product vision in any public materials I can see. At pre-seed, I give founders benefit of the doubt on distribution thinking, but the product architecture here looks like a middleware layer that gets commoditized rather than a platform that compounds.

The bull case I have to take seriously: the 58% blocked-action recovery rate is a genuinely interesting technical signal. Most guardrails products just block bad actions -- Salus blocks AND returns structured feedback enabling the agent to self-correct. If that recovery rate holds across diverse agent architectures, it's a meaningful differentiation. The cross-platform positioning (supporting OpenAI, Anthropic, LangChain, LangGraph, CrewAI) could become valuable if the agent ecosystem fragments rather than consolidates. And if enterprises standardize on multi-model agent deployments -- which seems plausible -- a vendor-neutral guardrails layer would fill a structural gap that no single LLM provider would serve. That's a version of the story where Salus becomes infrastructure, not middleware. But this requires two miracles: first, that external guardrails remain necessary versus native provider solutions, and second, that Salus achieves developer adoption before Fiddler AI's enterprise sales team and Guardrails AI's open-source community capture the market. That's compounding low-probability events.

The founders are Stanford CS students with relevant LLM research backgrounds -- Vedant Singh's work across five Stanford labs on LLM reasoning, NLP, and applied AI is directly germane. But these are first-time founders competing against teams with $100M+ in backing and multi-year head starts in enterprise relationships. No revenue signals, no customer names, no evidence of enterprise engagement beyond the YC network. The SDK exists and benchmarks are published, which shows execution capability, but the GitHub presence is sparse (one public repo with 5 stars from 2023, predating the current product). At pre-seed I don't penalize absence of traction, but I also don't see signals that would override the competitive and platform-risk concerns.

I'm passing. The timing is right for agent guardrails but the market is consensus, the moat is thin, and the competitive dynamics work against a two-person pre-seed team. When I invested in Coinbase, crypto infrastructure was "too unfamiliar" for most investors. When I backed PagerDuty, operations alerting was "too boring." Salus is building in a market that every infrastructure investor, every LLM provider, and every AI observability company is actively pursuing. The structural moment exists but it's already crowded with well-resourced competitors racing toward the same insertion point.
