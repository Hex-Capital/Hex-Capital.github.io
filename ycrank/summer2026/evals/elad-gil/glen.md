# Glen -- Elad Gil Evaluation

Nikos Dritsakos is a solo founder with one prior acquired AI-SaaS exit (SalesBop → SellWell/FliteHouse) shipping an org-wide agent-memory layer with zero disclosed revenue, zero named logos, and a "waitlist wave" production posture against Mem0's $24M and 48K+ GitHub stars — which makes Glen a credible-operator-without-a-curve, exactly the profile where my "false signal vs. just working" filter (TechCrunch, Nov 2025) treats silence as the signal.

**Perplexity** (2022) is the load-bearing analogy on the founder-shipping axis: Aravind cold-pinged me on LinkedIn and showed up a week later with the thing built, and a solo or near-solo operator with a demonstrated shipping rhythm can outrun a funded competitor in a re-opening category. Glen's Dritsakos clears the first half of that test — SalesBop wasn't a deck, it was a shipped AI-sales-coaching product that got acquired, and that is the kind of independent shipping artifact I weight over pedigree. Where Glen diverges from the Perplexity rung is that the dossier gives me no between-meeting cadence, no GitHub personal handle, no public commit history on @Glen-Web-App, and no demoable iteration loop — I'm pricing the prior exit, not an observed execution tempo, and that's a markdown not a markup.

**Harvey** (2023) is the template I keep coming back to on revenue velocity — "$3B → $5B → $8B in months" (TechCrunch, Nov 2025) is what "just working" looks like in dollars, and the inverse rule from Tim Ferriss Show #863 (April 2026) is blunt: "if you're an AI company and you're not seeing explosive growth quickly, something's fundamentally broken." Glen's dossier discloses no MRR, no pricing page, no Product Hunt, no Discord size, no follower counts on @try_glen, no customer logos — just "in production today" with waitlist waves (tryglen.com, 2026). Pre-seed gets a default discount, but in agent-memory the comparable cohort is loud — Mem0 publishes 186M Q3 2025 API calls and an AWS Agent SDK distribution deal (mem0.ai blog, 2026) — and the silence reads as absence, not stage.

The MCP standardization across Claude, Cursor, and Codex in 2025 is the catalyst question, and it compounds over 18 months in a specific way: if MCP becomes the default integration surface, every agent runtime needs a shared context layer, and Glen's per-observation access control architecture (tryglen.com, 2026) is the right primitive for that world. But the mechanism cuts both ways — Anthropic, Cursor, and OpenAI all have incentives to ship native memory inside their host runtimes, and LangMem is already framework-bundled inside LangChain (AgentMarketCap, Apr 2026). Eighteen months from now Glen either rides MCP as an index position on multi-agent orgs, or MCP-host vendors absorb the layer and Glen becomes a feature inside someone else's runtime. The catalyst is real; the position on top of it isn't yet defensible.

To flip Glen's index-position score a full tier, one of two things has to be true: either Glen ships a connector graph (PRs, docs, meetings, issues) so dense that switching costs compound per-customer in a way Mem0's vector-first SDK can't replicate, or Glen lands an MCP-host partnership that hard-wires it as the default shared store for one of Claude, Cursor, or Codex. Neither shows up in the dossier — what shows up is OAuth 2.1 and argon2id key hashing, which is table-stakes plumbing, not distribution. Without one of those two conditions, Glen is fighting Mem0 ($24M, 48K stars) and Letta ($10M at $70M post) for developer mindshare with one engineer.

**Color Genomics** (2015) is the single-miracle parallel I'd lean on: I architected Color as workflow on top of commodity sequencing precisely so the company didn't carry chemistry risk, market risk, and regulatory risk simultaneously. Glen is trying to run the same play — workflow on top of MCP — but the miracle count is higher than it looks: (a) MCP remains an open standard rather than getting absorbed into host runtimes, (b) cross-agent shared memory actually outperforms per-agent memory on task quality, and (c) a solo founder clears enterprise security review for code/PRs/docs/meetings access without SOC 2. That's not a science project, but it's not a single-miracle deal either.

My 36-month forecast: Glen reaches under $2M ARR and gets acqui-hired by Mem0, Letta, or a coding-agent host (Cursor or an Anthropic-adjacent runtime) for the connector graph and the per-observation access primitive, OR pivots to a narrower coding-agent context wedge (PRs + issues only) and survives as a small ARR business. The mechanism: solo-founder bandwidth against a six-source platform scope plus two funded competitors with distribution leads compresses the window before Glen can convert the MCP catalyst into a defensible index position.

The strongest bull case against my forecast is the **Perplexity** precedent — a near-solo founder with a demonstrated shipping rhythm rode a category re-opening past better-funded incumbents because shipping cadence beat balance sheet. Dritsakos has a prior acquired exit, MCP is a genuine standardization moment, and per-observation access control at recall is a real architectural bet that vector-first peers haven't made. The dismantling: Perplexity had a visible product loop the week I met Aravind; Glen's dossier gives me a waitlist, no public usage metrics, no shipping cadence I can observe, and Mem0 already owns the AWS Agent SDK distribution slot that Glen would need. **Verdict-flipping condition**: a signed named-enterprise logo with disclosed seat count, or $50K+ MRR within 90 days of YC Demo Day — either one would move Glen from "credible operator, silent curve" to "just working," and I'd write the check.
