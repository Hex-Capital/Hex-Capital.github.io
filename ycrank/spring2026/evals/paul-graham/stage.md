# Stage -- Paul Graham Evaluation

$334.5 million. That's the combined funding raised by CodeRabbit, Graphite, Greptile, and Qodo — Stage's four most direct competitors — all of whom shipped structured code review features before Stage existed. CodeRabbit alone has $15M ARR and 8,000 paying customers. Graphite got acquired by Cursor for $290M in December 2025. Stage enters this field as two people with no public traction, no visible users, and a website that didn't even render when researchers tried to access it. When I evaluate a startup, I start with the founders and treat the idea as mutable. But Stage's problem isn't the idea — it's that this idea has become everyone's idea simultaneously. "Better code review for AI-generated code" is the kind of thing that sounds obviously good, and if a good idea were obviously good, someone else would already have done it. Four very well-funded someones already have.

Dean Stratakos has the strongest founder-market signal in Stage's dossier. He led Five Rings' company-wide AI initiative, built an internal coding agent, and designed high-performance trading systems in C++ and Python. Charles Pan worked alongside him at Five Rings and was an early engineer at Yuzu Health, a YC-backed health insurance company. Both are Stanford CS — Stratakos with a master's in AI, Pan class of '22. These are technically capable people who can build their own product, which clears my minimum bar. Stratakos would have experienced firsthand the difficulty of reviewing code generated by the AI tools he built at Five Rings. That's real organic discovery — he lived in the future of AI-assisted development and noticed that review hadn't caught up. But here's my concern: every developer using Copilot or Cursor has noticed the same thing. The frustration is universal, which means the insight isn't proprietary. Compare this to Drew Houston forgetting his USB drive — a universal frustration, yes, but one where the solution (seamless file sync) required genuine technical infrastructure. Stage's "chapters" concept is a presentation-layer innovation. It reorganizes how diffs are displayed. That's useful, but it's the kind of feature GitHub could add as an optional view mode without breaking backward compatibility for their 100M+ developers.

Stage's "chapters" approach — breaking code changes into a narrative sequence rather than file-by-file diffs — addresses a real comprehension gap. I'll grant that. The bull case goes like this: Stratakos built an AI coding agent internally and saw the review problem from both sides (author and reviewer), Pan shipped product at a fast-moving startup, and together they noticed something the incumbents are too locked into their existing UX paradigms to fix. The $290M Graphite acquisition proves code review is a viable exit category. Maybe Stage's structured narrative approach is genuinely harder to bolt onto existing tools than it appears — maybe reorganizing the review experience is an architectural choice, not just a CSS change, and incumbents carrying years of technical debt can't adopt it quickly. If I squint, I can see a version of this where Stage becomes the Stripe of code comprehension — where the "chapters" abstraction is as fundamental a rethinking as Stripe's seven-line integration was for payments. But Stripe worked because Patrick Collison was willing to wrestle PCI compliance, bank partnerships, and regulatory complexity across dozens of countries. That was a schlep that repelled every other developer. "Making diffs more readable" doesn't repel anyone. It attracts everyone, which is exactly why $334.5M is already chasing it.

Charles Pan's LinkedIn still shows "Revel Street" rather than Stage, which is a small thing but the kind of small thing I notice. When the Airbnb founders were selling Obama O's cereal boxes to fund their company, there was no ambiguity about their commitment. I look for evidence that founders have done something extraordinary under constraint — built revenue before raising, shipped a prototype in a weekend, personally performed the service their software would automate. Stage's dossier contains zero such stories. Both founders have impressive résumés — Five Rings, Apple (three internships), Citadel, Stanford varsity athletics — but impressive résumés describe what institutions selected you for, not what you built when nobody was selecting you. The absence of resourcefulness stories at a pre-seed stage isn't damning by itself, but it means I'm relying entirely on credentials rather than demonstrated behavior, and credentials are a weak proxy for the quality that actually matters.

Stage fits my "hackers building for hackers" pattern — developer tools built by developers — but that pattern works when the tool addresses a problem that's both painful and underserved. Stripe addressed payments infrastructure nobody wanted to build. Replit addressed accessible programming environments that incumbents actively neglected. Stage addresses code review comprehension in a market where CodeRabbit ($88M raised), Qodo ($120M raised), and Greptile ($45.5M raised) are all aggressively building AI-powered review tools with features that overlap Stage's value proposition. The dossier itself flags that the "chapters" concept is "a UX innovation rather than a deep technical moat, making it reproducible by any competitor with front-end engineering resources." That's exactly right. Stage's differentiation window is short, and the competitors have the engineering teams and user bases to close it.

The one verifiable condition that would flip me to Invest: evidence that a meaningful number of engineers — say fifty at three or more companies — are using Stage daily and refusing to go back to standard diff views. That would tell me the "chapters" abstraction has genuine switching costs and isn't just a nice-to-have. Obsessive early users using a crappy version one would change everything about this evaluation, because it would mean the UX insight runs deeper than I'm giving it credit for. But that evidence doesn't exist in the dossier, and Stage's website didn't even load for researchers. The founders are technically qualified, the problem is real, and the organic discovery through Five Rings is legitimate. But Stage is entering a consensus-good market with a replicable UX innovation against entrenched, well-capitalized competitors, and I see no evidence of the extraordinary founder qualities that would make me bet on them overcoming those odds. The two dimensions that lock this verdict are Idea That Repels (6/20 — this idea attracts everyone, which is my clearest negative signal) and Founder Resourcefulness (13/30 — solid credentials but no stories of creative problem-solving under constraint, which is the signal I trust most).
