# Ashr -- Sam Lessin Evaluation

Ashr is the archetype of the deal I pass on fastest: a sane idea built by competent people in a crowded market. My entire framework is calibrated to find the opposite -- a sane person with an insane idea. "AI agents need better testing" is not an insane idea. It's a consensus observation. The LangChain State of Agent Engineering report has already quantified the gap. At least four funded startups -- Janus, Coval, Maxim AI, LangWatch -- are attacking the same problem, plus open-source alternatives like Langfuse. When I look at a market where five competitors are already operating at pre-seed, what I see is that the "insane idea premium" has been completely arbitraged away. The founders are competent Berkeley engineers -- Rohan Kulkarni has a small exit with Ask Geri, Shreyas Kaps has two years building AI agents -- but competence pursuing a consensus opportunity is exactly the combination that produces median outcomes. Neither founder built infrastructure at scale or brings the kind of credentialed depth that makes me believe they'll win a crowded knife fight.

The structural moat problem here is severe, and it's the exact problem I've been writing about publicly for two years. Ashr is pure software testing pure software. There is no physical-world component. There is no protocol position. There is no proprietary dataset that compounds over time -- the dossier itself states "No defensibility signals found in public sources at this stage." The switching costs are minimal: swap out one SDK for another. When I apply my AI kill test -- does this business get stronger or weaker as software gets cheaper? -- the answer is unambiguously weaker. As LLMs get more capable, generating synthetic test journeys becomes trivially easy for any developer with API access. Worse, the platforms Ashr depends on have every incentive to bundle evaluation natively. OpenAI, Anthropic, and LangChain are all building toward integrated agent development environments. This company is building a feature, not a business.

The bull case would require believing that agent testing becomes so complex and so mission-critical that it demands a standalone, specialized platform with deep institutional switching costs -- the way Datadog carved out observability from generic monitoring. If AI agents proliferate across every enterprise workflow, the quality assurance layer could become genuinely critical infrastructure. Ashr's multimodal breadth (voice, image, file generation) is broader than most competitors and could serve as a differentiation wedge. If the founders could accumulate proprietary scenario libraries and evaluation frameworks tuned to specific industries -- healthcare agent testing, financial agent testing -- they might build something defensible. But "might build switching costs over time" is a hope, not a moat. And the Datadog analogy actually works against them: Datadog itself could add agent simulation features more easily than Ashr could build Datadog's distribution.

This company fails my "cherry on top" test completely. Remove AI from the equation and there is no business whatsoever -- the entire value proposition exists only because AI agents exist. That's the opposite of what I look for. I want businesses where AI is non-obviously helpful, where the core economics work without it. When the AI angle is this obvious, the market has already priced it in. The $0 pricing with no visible paid tiers, the customer base weighted toward academic users (UC Berkeley, Stanford) rather than paying enterprises, and the absence of any revenue signal all confirm that this is still in validation phase with no demonstrated willingness to pay. I need either an infinity story or a downside cushion. I see neither here.
