# Memory Store -- Naval Ravikant Evaluation

Memory Store is a deep-specific-knowledge builder bet wrapped in a single-sided $150/seat B2B subscription -- Diwank Tomer and Ishita Jindal spent four years shipping Julep (6.6K stars, open-source agent orchestrator with persistent memory primitives) and are now selling that same expertise as a turnkey team workspace, which means the founder lens is strong but the leverage architecture is weaker than my portfolio anchors require. The signature pairing is high specific knowledge against low network compounding: this is a Perplexity-shaped single-tenant utility carried by founder-product-market fit rather than by the compounding-user-value math that funded **Uber** (2010) or **Stack Overflow** (2008).

The highest dimension here is specific knowledge, and the load-bearing analogy is **Stack Overflow** (2008) -- Atwood and Spolsky were practicing developers writing for developers, and the seed gate cleared on the lived problem more than on any business-model story. Memory Store's parallel is real: Diwank has been building AI products since 2012, the 2013 Thiel Fellowship dates the obsession, and Julep's 6.6K stars are not credentialism but shipped artifact in the exact primitive -- persistent memory for agents -- the new company sells. Where Memory Store diverges from the Stack Overflow rung is the audience-of-self test: Atwood was the user, but the Memory Store team is shipping a *teams* product on a $150/user/month tier to companies they have not themselves operated inside, and the Julep backend shutdown on 2025-12-31 shows the prior six-thousand-star community did not convert to commercial pull. Specific knowledge is present at top-quartile depth; specific knowledge of the *buyer* is not the same thing.

**Perplexity** (~2022) is the failure-mechanism comparable for Memory Store's lowest dimension -- a single-sided foundation-model-adjacent app where the Nth user does not compound the (N-1)th user's value. I waived that test for Aravind Srinivas because the specific-knowledge gate was extreme, the API-as-distribution was permissionless, and the pre-consensus AI-cycle timing was emphatic. Memory Store fits the same shape -- each team's ingested Gmail/Slack/Granola corpus is private to that team, no cross-tenant network forms, and the "team memory graph" lock-in is asserted rather than demonstrated -- but Memory Store does not collect the same three offsetting passes simultaneously: distribution leans on MCP-host permission (Claude, ChatGPT, Cursor must keep the plugin slot open), and we are not pre-consensus on memory layers anymore with Mem0 already at $24M Series A and Letta at $10M from Felicis.

The leverage architecture is the entry-point check, and Memory Store is a mixed read: the MCP-plugin distribution surface across Claude, ChatGPT, Cursor, Raycast, and Linear is genuine code-leverage -- one integration shipped, many client agents activated -- but the monetization motion is a single $150/seat enterprise plan billed monthly with no free tier surfaced in the dossier. Over 18 months that compounds the wrong way: each new MCP host adoption is permissionless reach, but each new $150/seat closed deal pulls the company toward labor-leverage selling (SDRs, AE motion, customer success) because per-seat enterprise pricing does not self-serve at that price point. The architecture is half-Twitter, half-Salesforce; the half that grows fastest determines which company Memory Store becomes.

Market and pivot-room would have to fail spectacularly for Memory Store to score lower here -- "memory layer for AI agents" is platform-shaped, not vertical-niched, and the dossier names at least three adjacent pivot vectors (developer SDK competing with Mem0, enterprise search competing with Glean, consumer memory competing with nothing yet). For this dimension to flip a tier downward, the team would need to lock into a single sub-vertical like "memory for sales teams" or "memory for legal" -- the dossier shows the opposite, integrations spanning Gmail, Slack, Granola, Fathom, Linear, OpenCode, which is platform-positioning. The pivot-room safety margin holds.

The founder trio on Memory Store reads like a mid-rung **Notion**-adjacent profile rather than a flag: Diwank's prior product reportedly powers phone ordering for Domino's, Wingstop, and Buffalo Wild Wings (operator credibility, shipped revenue elsewhere), Ishita led products at Jar and Lenskart at millions-of-users scale, and the Julep backend wind-down on 2025-12-31 was a clean stop rather than a contested unwind. Four years of shared operating history before Memory Store is the variable I weight most -- co-founder selection is the most important decision, and they made it once already and stayed together through a product sunset. No volunteered statements about honesty, no logos-as-thesis pitch, no 50/50 unresolved-CEO red flag visible.

Thirty-six-month forecast: Memory Store reaches $2-5M ARR by mid-2028 on roughly 100-300 paying teams at the $150/seat tier, then gets acqui-hired by an MCP host -- Anthropic, Cursor, or a memory-layer roll-up by Notion or Glean -- as the team-shared-context feature gets absorbed into the agent host rather than living as a standalone plugin. The mechanism is the platform-dependency risk the dossier names explicitly: Memory Store's value proposition routes through Claude/ChatGPT/Cursor adoption, and whichever host commits hardest to native team memory eats the standalone layer. Specific knowledge plus clean founders plus platform-shaped market produces an acqui-hire floor, not a 1,000x outcome.

The strongest bear case against even that modest forecast is the Julep precedent: Diwank and Ishita already ran one open-source memory company to 6.6K stars and could not convert it to commercial pull, shutting the backend on 2025-12-31 before Memory Store launched. The bear reads this as a *commercial-conversion* deficit dressed up as a product reset -- same founders, same primitive, new wrapper, same outcome. That sting is real, and the Julep shutdown is the single most negative datum in the dossier. What dismantles it partly is that Memory Store is the *learning* from Julep -- developer SDKs at 6.6K stars do not monetize, so they moved up-stack to a packaged teams product with a price tag, which is the correct iteration. The verifiable condition that flips my verdict: if Memory Store does not show 50+ paying teams at the $150 tier within twelve months of YC Launch (so by roughly May 2027), the Julep pattern is repeating and the acqui-hire floor itself is at risk. At this score and stage I do not write a check, but I keep the founders on the watch list for the next round at materially better evidence.
