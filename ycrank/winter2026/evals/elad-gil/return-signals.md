# Return Signals -- Elad Gil Evaluation

The competitive dynamics here concern me more than anything else in this dossier. Attentive is at $500M ARR. Klaviyo is at $1.2B ARR. Loop Returns has raised $176M. Return Signals is building a product -- proactive post-delivery conversational SMS -- that sits in the gap between these incumbents. The question I always ask in this situation: is this gap a durable market, or is it a feature that one of those incumbents ships in a quarter? Attentive already has SMS infrastructure, e-commerce integrations, and thousands of brand relationships. Adding a post-delivery AI conversation module is an engineering project, not a strategic pivot. Loop Returns already owns the returns workflow and could layer AI outreach on top of their existing portal. When I map the competitive structure here, I see a product that creates value in a space where well-funded incumbents have every incentive and capability to absorb it. That's the "market-ending move" dynamic working against you, not for you.

The structural moment question is where this falls short of my framework. What changed in the last 12-24 months? The dossier infers three things: LLMs crossed a conversational quality threshold, Emotive got acquired creating a vacuum, and SMS adoption grew 40% year-over-year. The LLM threshold is real but it's the same "why now" for every AI application company I see -- it's necessary but tells me nothing about this specific market's inflection point. Emotive's acquisition is a competitive opportunity, not a structural shift. SMS growth is a secular trend. What I'm missing is a market-specific inflection -- something analogous to how Google's Project Maven cancellation created a structural vacancy for Anduril, or how the GPT-3 to GPT-4 capability jump specifically unlocked legal-grade accuracy for Harvey. "LLMs got better" doesn't explain why proactive returns prevention via SMS is a venture-scale opportunity now versus a nice feature that incumbents will bundle.

The product-to-distribution trajectory is where I have the least conviction. Return Signals' core interaction is episodic: a customer receives a delivery, gets an SMS check-in, has a conversation, and the interaction ends. Compare that to Gusto, where payroll is recurring and mandatory -- that persistent touchpoint became the distribution channel for HR, benefits, compliance, and tax products. Or Stripe, where the payment API integration sits in every transaction, enabling Atlas, Billing, and Treasury to flow through the same developer relationship. Return Signals collects useful data (sizing feedback, return reasons, sentiment), and I can see how that could theoretically power a merchandising intelligence product. But data collection is not distribution -- it's an input to a product you'd still need to build and sell separately. The path from "post-delivery SMS concierge" to "platform that distributes multiple products through a brand relationship" requires a strategic leap that I don't see evidence the founders have mapped.

The strongest bull case: e-commerce returns are a genuinely expensive problem -- US returns exceeded $200B in 2023, and apparel return rates run 30%+. If Return Signals' metrics hold at scale (70% conversion of return intents to exchanges, 5-10% margin uplift), the ROI story is compelling enough that brands would pay meaningfully for it. Ilya's background building SmartTriage at Kaiser Permanente -- one of the largest patient-facing conversational AI deployments -- is directly transferable technical expertise. The pattern of building high-stakes conversational AI in healthcare and applying it to a lower-stakes domain (e-commerce) is actually smart: the technical bar is lower, the regulatory burden is lighter, and the deployment is simpler. If Attentive and Klaviyo are structurally unable to build this because their revenue model scales with marketing message volume (which is orthogonal to returns prevention), then the business model conflict cited in the dossier could create enough incumbency drag to give Return Signals a real window. That's the version of this story where narrow vertical focus plus a founder with genuine conversational AI deployment experience creates a defensible niche. For this to work, the team would need to prove that the domain-specific data moat (brand-level sizing curves, return-reason patterns) compounds fast enough that the product becomes meaningfully better than what Attentive could build generically.

But I don't think that's the most likely outcome. The data moat doesn't exist yet at pre-seed. Neither founder has e-commerce or fashion/retail domain experience -- Ilya's background is healthcare AI, Alejandro's is proptech business development. The co-founders have no visible shared work history or overlap in their professional backgrounds, and e-commerce returns prevention is a domain neither has operated in. When I apply the single-miracle test, there are two obstacles that need to clear: the product has to work at scale (plausible given Ilya's deployment background), AND the company has to achieve distribution before incumbents absorb the feature. That's two miracles, and the second one is a race against companies with 100x more resources and existing brand relationships. The claimed metrics are encouraging but lack independent validation -- no named customers, no revenue figures, no third-party confirmation. This feels like a well-executed feature looking for a company to become, in a market where the structural moment isn't differentiated enough to create the asymmetric return profile I need.
