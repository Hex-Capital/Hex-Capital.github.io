# Zenbu -- Naval Ravikant Evaluation

Rob Pruzan ships developer tools and his last one — React Scan — collected 22k GitHub stars and 800k weekly installs before he ever pitched Zenbu, which is the signature that matters here: this is a builder-product-by-a-builder with top-quartile specific knowledge welded to a single-sided IDE that lacks the compounding-user-value math my 10,000x bets need. Zenbu is Perplexity-shaped in my ledger — strong on the unstated builder-tools-for-builders pattern, weak on criterion #2 — which means the exception math has to do the work, and at 222 zenbu.js stars against Cursor's $2B ARR and 40M users, the math isn't yet doing it.

Stack Overflow (2008) is the load-bearing analogy on specific knowledge — Atwood and Spolsky were practicing developers writing for developers, and the seed cleared on that alone before any monetization existed. Pruzan's ex-Vercel-Next.js-core-team line plus React Scan's 800k weekly installs is the same template applied one cycle later: he is a direct end-user of coding agents building the IDE he himself wants, not an ex-McKinsey PM who identified a $9.46B TAM. Where Zenbu diverges from the Stack Overflow rung is artifact-to-thesis distance — Atwood had already built CodingHorror's audience around the exact developer-Q&A problem he then solved, while Pruzan's React Scan audience is adjacent to agent-IDEs, not native to them; the specific knowledge transfers but doesn't compound automatically.

Perplexity (2022) is the rung that names Zenbu's structural weakness — a single-sided developer tool where the Nth user does not directly make the (N-1) prior users more valuable, and Aravind Srinivas only cleared my Marketplace criterion via the explicit Known Exception because criteria #1, #3, and category-timing all hit top quartile simultaneously. Trace the pattern in Zenbu's dossier: the plugin ecosystem in ZenbuJS *could* compound switching costs if third-party plugins accumulate, but the dossier flags it as "unproven now given solo-founder stage and 222-star framework repo." This is a forward-option, not a present mechanic. Perplexity got the waiver because three gates cleared emphatically; Zenbu has one (specific knowledge) clearing emphatically and the rest at mid-range, so the exception math does not yet trigger.

Permissionless leverage on Zenbu compounds through two channels over the next 18 months: zenbu.js as open-source code-leverage (currently 222 stars, the wrong order of magnitude for developer-pull) and Pruzan's media-leverage off React Scan (15.8k stars on the canonical repo, 4,431 X followers — modest for a media moat). The mechanism that would matter is the React Scan install base converting to Zenbu trial without any paid acquisition; if 1% of those 800k weekly installs touch Zenbu, the GitHub Copilot/Cursor/Windsurf incumbents cannot replicate it because Pruzan owns the upstream relationship. The mechanism that would kill it is incumbent feature absorption — Cursor at $2B ARR can ship plugin-editability in a quarter, and OpenAI-owned Windsurf has $3B of acquisition capital backing Cascade — so Zenbu's leverage compounds only if it crosses 5k+ zenbu.js stars before either ships hackability natively.

Counterfactual for criterion #4 — the market score flips up a tier only if Pruzan articulates a pivot vector off "Pi-native IDE" before Pi's external maintainer (badlogic/pi-mono) introduces upstream friction. The AI Code Tools market is $9.46B in 2026 projected to $22.2B by 2030 at 23.8% CAGR, which is platform-shaped and absorbs pivots; what the dossier does not show is whether Zenbu has decoupled its identity from a single third-party agent runtime. "Niche markets don't work because the first idea never works" — Zenbu's first idea is Pi-native, and the second idea has to exist on day one of the seed.

AngelList (2010) is the closest single-anchor parallel on the founder-trio dimension — solo-leader architecture, builder-for-builders identity, clean integrity reads in the dossier with no contested exits or self-promotion about ethics. Zenbu's founder profile carries the same quiet signal: Pruzan's GitHub shows 89 followers and shipped artifacts named (react-scan, react-scratch at 126 stars), not pedigree-as-thesis. Solo at YC is a real execution concern against Cline's $32M war chest, but I would not score the integrity leg down for it — the 10-year-partner test reads pass on the available evidence.

36-month forecast: Zenbu reaches $1–4M ARR as a power-user hackable IDE for the agent-orchestration niche and is acquired by Vercel, Replit, or a tier-two infra buyer in the $30–80M range as Cursor and OpenAI/Windsurf absorb multi-agent orchestration as a feature and Superset (same YC P26 batch) splits the remaining oxygen. The mechanism producing this outcome is the geometry of single-sided builder-products without compounding-user-value architecture against well-capitalized incumbents — exactly the Perplexity shape minus the category-timing tailwind, because AI coding tools are now consensus, not pre-consensus.

The strongest bull case is that Pruzan's React Scan distribution (800k weekly installs) is a dormant developer-channel asset that ignites the Stack Overflow developer-pull pattern once zenbu.js plugin authorship crosses critical mass, and the "hackable" thesis becomes the Emacs-of-agent-IDEs lock-in that Cursor's closed architecture structurally cannot ship. This stings because builder-tools-for-builders is exactly my unstated pattern — Stack Overflow, Notion, Replit, AngelList, Quill Notes — and I have written this check before on thinner evidence than 22k stars and a Vercel-core pedigree. What dismantles the bull case is the same-batch Superset overlap and the Pi dependency: when two YC P26 companies pitch "IDE for the agents era," the category is not pre-consensus, and when the core agent is maintained by an external open-source author, the leverage architecture has a permission gate inside it. Verifiable flip condition: zenbu.js crosses 5,000 stars and 40+ third-party plugins with measurable installs within 12 months — at that threshold the compounding-user-value math activates, the Perplexity exception triggers, and I write the check at the next round.
