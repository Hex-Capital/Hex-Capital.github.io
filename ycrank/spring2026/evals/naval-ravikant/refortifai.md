# Refortifai -- Naval Ravikant Evaluation

Microsoft released the Agent Governance Toolkit as free open-source software under MIT license on April 2, 2026 — the same quarter Refortifai entered YC. Seven packages, five languages, sub-0.1ms p99 latency, covering all ten OWASP agentic AI risks, integrating with LangChain, CrewAI, and Google ADK. When the platform owner gives away the exact product you're building, you're not selling software — you're selling implementation services. That's labor leverage, not code leverage. Refortifai's core use case as described on the YC page — intercept tool calls, enforce governance policies, allow or deny — is architecturally the kind of pattern that becomes a feature inside every agent framework, not a standalone company. Innovation without a moat is consumer surplus. All the value flows to the customer and eventually to the platform provider. Capsule Security raised $7M in the same month. Noma Security has $132M and reports 1,300% ARR growth. The field is not just crowded — it's converging on commodity.

Refortifai's most revealing signal is the gap between its website and its YC page. The website describes "anti-cheat for AI models" — weight obfuscation before distribution, secured runtime keeping original values unexposed, never on disk or in RAM. The YC page describes "a security agent that monitors your agent" — tool-call interception and governance policies. These are fundamentally different products serving different buyers against different threat models. At pre-seed, a company that hasn't resolved which problem it's solving is a company whose founders identified two adjacent opportunities and are hedging. Hedging is agent behavior. Principals don't hedge — they're obsessed with one thing. Refortifai's product ambiguity is the clearest signal in this dossier that the founders are pattern-matching to what's fundable rather than building from compulsion.

Atman Kar's background is the closest thing Refortifai has to specific knowledge. RTL digital design at Texas Instruments building automotive RADAR hardware accelerators, CPU RTL internship at Tenstorrent, an MS from NC State — that's genuine low-level systems expertise. His GitHub confirms systems orientation: a 6502 emulator in C, an OS for i386, Radon transforms in Rust. But here's the problem: that specific knowledge maps to the model-weight obfuscation product, not the agent runtime security product. The product Refortifai is pitching to YC — monitoring agent tool calls, enforcing policies — requires application security expertise, threat research experience, understanding of prompt injection attack vectors. No founder on Refortifai's team has a confirmed background in cybersecurity. Sayan Mitra's background is largely invisible in public data; a possible Zscaler connection remains unconfirmed. Rithik Jain transitioned from economics to computer science at UW-Madison — breadth, not the depth that constitutes a moat. Refortifai's specific knowledge points in one direction while the company walks in another.

The bull case deserves honest construction. Refortifai could argue: agents are proliferating at Gartner's projected 40% enterprise penetration by 2026, security is a must-have, and CalypsoAI's $180M acquisition by F5 proves the exit path. The model-weight obfuscation technology — if it delivers "negligible overhead" versus homomorphic encryption at "full inference speed on any hardware" — could represent a genuinely hard technical problem where Atman Kar's hardware expertise creates real IP defensibility. Three friends building together (Rithik's LinkedIn describes Sayan and Atman as friends) is a stronger co-founder foundation than most YC teams. The known exception I'd check: Belfort, which I backed alongside Jeff Dean — fully homomorphic encryption chips creating new forms of permissionless leverage through hardware. Does Refortifai's weight obfuscation qualify? It's the right category of bet. But Belfort had Jeff Dean's co-investment as a parallel diligence signal validating deep technical claims. Refortifai has no visible co-investor of that caliber, no patents in public records, and critically — they appear to be pivoting away from the product where their specific knowledge actually lives toward the one where Microsoft just gave away the answer for free.

Refortifai's leverage architecture fails the compound interest test. A policy engine that intercepts and allows or denies tool calls doesn't get smarter from usage unless Refortifai builds a proprietary threat intelligence layer from cross-customer attack data — a data network effect. But building that flywheel requires exactly the security domain expertise the team doesn't visibly possess. Compare this to Perplexity, where Aravind Srinivas's specific knowledge from Google Brain and OpenAI was the engine that made each query compound into better answers. Refortifai has no visible equivalent — no founder who spent years thinking about how agent security should work, no prior security research, no open-source security contributions. The three-person team is appropriately lean, and the YC selection provides some validation, but team density without domain-specific capability is just a small group of generalists. Refortifai's access democratization angle — making agent security available to companies that can't afford Noma's enterprise platform — is real but generic. Every SaaS security startup makes this claim.

One verifiable condition would flip me: if Sayan Mitra's unconfirmed Zscaler background turns out to be senior security engineering with years of hands-on threat research, and if Refortifai has already built proprietary detection capabilities that go meaningfully beyond Microsoft's open-source toolkit — not incremental UI polish, but novel attack detection that requires specific knowledge to build — I would re-evaluate. The compound of confirmed security expertise plus technical differentiation would change the specific knowledge score from weakness to potential strength. Absent that, the two dimensions that lock Refortifai into Pass territory are specific knowledge productization and permissionless leverage architecture. The founders' visible expertise is misaligned with the product they're pitching, and the product pattern is commodity enough that the world's largest software company gave it away for free three weeks before demo day. You can't build a lasting company on leverage someone else hands out at no cost.
