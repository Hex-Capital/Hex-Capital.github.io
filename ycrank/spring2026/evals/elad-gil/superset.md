# Superset -- Elad Gil Evaluation

The first thing I notice about Superset is that it's building on top of an actively contested platform layer where the infrastructure owners have every incentive to absorb the functionality. Anthropic has already shipped native multi-agent orchestration for Claude Code. This is the exact red flag I watch for: when you're building an application on a platform layer that hasn't been won yet, you face existential platform risk. Stripe succeeded because payment networks were stable infrastructure -- nobody at Visa was shipping a developer-friendly API. Superset is building a coordination layer for AI coding agents while those agent providers are in a land grab, actively expanding their own surface area. The analogy isn't "picks and shovels during a gold rush" -- it's more like building a multi-mine management dashboard while the mine owners are building their own dashboards.

Let me start where I always start: market structure timing. The structural shift here -- CLI-based AI coding agents emerging in 2025 -- is real. Claude Code, Codex CLI, Gemini CLI have created a genuine new category. But this is the most consensus-obvious market in technology right now. Cursor is valued at $29.3 billion. Every developer tools investor on Earth is looking at AI coding infrastructure. When I backed Coinbase, crypto was dismissed by mainstream investors. When I wrote the seed check for Perplexity, AI search was considered a dead category. Superset is entering a market where the structural shift is on the front page of every tech publication. The specific niche of multi-agent orchestration is already being addressed by Conductor (YC S24), Claude Squad (6.6k GitHub stars), and the agent providers themselves. There's no timestamp mismatch between reality and consensus here -- everyone sees this.

The infrastructure-layer test is where Superset struggles most. An infrastructure company builds something that other businesses integrate into core workflows, creating compounding switching costs. Stripe became the payments API; Figma became the design system of record. Superset wraps existing CLI tools in a desktop application with Git worktree isolation and a diff viewer. The core value proposition -- running multiple agents in parallel without merge conflicts -- is a workflow convenience, not an integration layer. Users supply their own API keys, Superset doesn't sit in the data flow or the billing flow, and there's no API that third parties build on top of. If Claude Code ships robust native parallelism tomorrow, a developer switches away from Superset with zero migration cost. Compare that to ripping out Stripe from your payment stack. The 8,000 GitHub stars and 595 forks represent real developer interest, and that's genuine signal. But community adoption of a free tool doesn't automatically translate into infrastructure lock-in.

The single-miracle test reveals compounding risk. For Superset to win, multiple independent conditions must hold simultaneously: the AI agent market must remain fragmented enough that an agent-agnostic orchestration layer matters (if one agent dominates, that agent ships its own orchestration); IDE vendors like VS Code, Cursor, and JetBrains must not add native multi-agent management as a feature (these companies have millions of users and strong incentive to add this); and the standalone multi-agent orchestration workflow must become standard enough to sustain an independent product company with venture-scale economics. That's at least three miracles, and one of them -- agent providers not shipping native orchestration -- is already failing.

The strongest bull case deserves serious consideration. Developer tooling has a pattern where focused, independent tools beat bundled solutions: Git beat integrated version control in IDEs, Docker beat VM management suites, even tmux persists despite terminal emulators adding tab management. If the AI agent market fragments across five or six viable CLI agents -- and there's a reasonable argument it will, given different model strengths for different tasks -- then the orchestration layer becomes genuinely valuable. The agent-agnostic positioning would be the right bet in that world, because no single agent provider would build great support for their competitors. Kiet Ho's track record building Onlook to 25,000 GitHub stars demonstrates real ability to ship open-source developer tools that developers actually adopt. And the three-CTO founding team brings deep technical credibility across multiple domains. In this scenario, Superset becomes the control plane for multi-agent development workflows, and 8,000 stars in five months is the early traction signal. I take this scenario seriously -- but it requires the fragmentation miracle AND the feature-absorption-resistance miracle to both land, and that's where my framework says the odds compound against you.

On the founders: three ex-YC CTOs is solid technical DNA, and the fact that all three left their previous YC companies to pursue this suggests real conviction about the opportunity. But the dossier doesn't clarify the decision-making hierarchy -- three CTOs with no designated CEO is the equal co-founder structure I flag as a risk for scaling. I've seen this pattern: technical co-founders who are brilliant at building the v1 product but struggle when the company needs to make hard calls about pricing, enterprise sales strategy, or organizational design. That said, at pre-seed, the technical execution capability matters more than the org chart, and this team clearly ships.

The margin architecture concerns me. The BYOK model -- where users supply their own API keys and Superset doesn't proxy calls -- means the company is explicitly outside the revenue flow. The free tier is live, enterprise pricing is undefined. For a developer tool to build venture-scale economics, it needs to either embed in workflows deeply enough to charge meaningful subscription fees or capture usage-based revenue that scales with value delivered. Right now I see a free productivity tool searching for a business model, operating in a category where the most powerful competitors (Cursor, GitHub Copilot) have already established developer willingness to pay at the agent layer, not the orchestration layer. This is a pass for me -- the market is too consensus-obvious, the positioning is application-layer rather than infrastructure-layer, and the company needs multiple simultaneous miracles to avoid being absorbed by upstream providers.
