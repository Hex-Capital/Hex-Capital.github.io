# Stage -- Vinod Khosla Evaluation

$334.5 million. That's the combined funding already deployed into Stage's four closest competitors — CodeRabbit ($88M), Graphite ($81M, acquired by Cursor for $290M), Greptile ($45.5M), and Qodo ($120M). Stage enters this market as a two-person team whose core differentiation is organizing diffs into "structured chapters." The dossier itself identifies this as "a UX innovation rather than a deep technical moat, making it reproducible by any competitor with front-end engineering resources." I don't need to spend long on this. When your competitive advantage can be replicated by a front-end engineer at any of four well-capitalized incumbents, you don't have a competitive advantage. You have a feature request on somebody else's roadmap.

Stage's consequence magnitude is where my evaluation starts and effectively ends. If Stage fully succeeds — if every engineering team on earth adopts structured chapters for code review — what changes? Software engineers read pull requests slightly faster. Code review becomes marginally less painful. This affects perhaps 30 million developers worldwide, and it affects them in a way they'll barely notice by next quarter. Nobody's healthcare gets cheaper. Nobody's energy costs collapse. Nobody who couldn't previously access a basic human need suddenly can. Stage is precisely the category I've described before: a productivity tool for a specific profession where success means a moderately profitable SaaS company. The consequence of Stage succeeding is that a venture fund makes 3-5x its money. That's fine for most investors. It's not fine for me.

Dean Stratakos built an internal coding agent at Five Rings and led company-wide AI initiatives — that's genuine domain expertise in exactly the problem Stage is solving. Charles Pan shipped code at Yuzu Health and Five Rings. Both are Stanford CS, both varsity athletes, both worked at the same quantitative trading firm. The resumes are impressive. But impressive resumes are exactly my low-scoring signal for founder evaluation. I don't see evidence of what surprised them, what they pivoted on, or how their thinking evolved during YC. I see two well-credentialed engineers who noticed a real problem at their previous employer and built a clean solution. That's competent execution, not first-principles rethinking of how humans and machines collaborate on code. Stratakos's three Apple internships and Citadel experience demonstrate consistency, not the irreverent outsider energy I look for — contrast this with Peter Beck building rocket engines in a New Zealand garage or a 19-year-old dropout at Mach Industries taking on Lockheed Martin.

Let me construct the strongest bull case an advocate would make. First: the "why now" is real. AI-generated code is creating a genuine comprehension crisis — when neither the author nor the reviewer fully understands what Cursor or Claude Code produced, structured presentation becomes critical infrastructure, not a nice-to-have. Second: Cursor's $290M acquisition of Graphite proves code review is a strategic acquisition target for AI coding platforms. Stage could be built to be acquired. Third: both founders have direct experience with the exact problem — Stratakos literally built the internal coding agent that generates the kind of opaque code Stage helps reviewers understand. Fourth: CodeRabbit reached $15M ARR and 8,000+ paying customers, proving willingness to pay in this category. The problem is that every piece of this bull case describes a company designed to generate a moderate outcome — an acqui-hire or a feature acquisition. Stage isn't attempting to replace code review. Stage isn't attempting to make software engineering 10x cheaper. Stage is attempting to reorganize how humans look at diffs. That's the distinction between a company I fund and a company I wish well.

Stage's "chapters" approach fails my non-linearity test decisively. Reorganizing the presentation of a diff is a 1.3x improvement, not a 10x improvement. The technology is front-end engineering — there's no novel algorithm, no proprietary data flywheel, no science breakthrough. Compare this to Rocket Lab's Rutherford engine: electric turbopumps and 3D-printed combustion chambers created a step-function cost reduction in small satellite launch that couldn't be replicated by bolting a feature onto an existing rocket. Stage's chapters can be replicated by bolting a feature onto GitHub's existing PR interface. GitHub's constraint — that restructuring diffs would break workflows for 100M+ developers — is real but weak: they can ship it as an optional view mode, exactly as the dossier notes. Stage's path to radical affordability is similarly absent: code review is already free, bundled into GitHub and GitLab. Stage would charge money for a slightly better version of something that costs nothing today. That's the opposite of my thesis.

One verifiable condition would flip me to Invest: if Stage demonstrated that their structured review approach caught categories of bugs that traditional diff review systematically misses — not 40% faster review, but entire classes of defects that become visible only through narrative restructuring — and they had data proving it from real engineering teams. That would transform Stage from a UX improvement into a quality infrastructure layer with genuine consequence magnitude. Without that evidence, Stage is a well-built solution to a real but incremental problem, entering a market that has already been capitalized at scale by competitors with years of head start. I don't fund features. I fund futures.
