# Orthogonal -- Vinod Khosla Evaluation

Stripe has already processed 50 million transactions through its x402 protocol for agentic payments. Stripe -- with millions of existing developer relationships, a $91 billion market cap, and partnerships with OpenAI and Coinbase -- has declared this exact category as strategic. When I backed Juniper against Cisco, Cisco's CTO dismissed TCP/IP routing as irrelevant. That dismissal was the signal. Here, the incumbent isn't dismissing the category -- it's building it. Orthogonal is not attacking a problem that Stripe considers impossible. It's building a curated convenience layer adjacent to a problem Stripe is already solving with overwhelming distribution advantages. That's a fundamentally different competitive dynamic than the one that produces 2,500x returns.

Let me start where I always start: if Orthogonal succeeds completely, what changes in the world? Developers manage API keys through one platform instead of twenty. Billing gets consolidated. Discovery gets easier. This is a genuine pain point -- I don't dispute that. But the consequence of full success is a useful middleware business, not an industry transformation. Nobody's life changes fundamentally. No trillion-dollar system becomes obsolete. Compare this to what Square did: Square didn't aggregate existing payment terminals -- it made the entire traditional POS infrastructure irrelevant by giving every small business a card reader. That was structural disruption. Orthogonal is aggregating existing API access into a nicer wrapper. The world with a successful Orthogonal looks almost identical to the world without it -- developers just have slightly less operational overhead.

The bull case deserves genuine engagement. The agentic AI market is on a real exponential curve -- $7.5 billion today, potentially $200 billion by 2034. If AI agents become the dominant way software consumes APIs, whoever controls the access and payment layer sits on a massive toll road. You could argue this is analogous to Stripe's own origin: a simple developer tool that became the financial infrastructure for the internet. Maybe Orthogonal starts with 16 curated APIs and becomes the default agent-to-service layer for millions of API calls. The MCP standard just launched in late 2024, meaning the infrastructure patterns haven't ossified yet. And the founders bring relevant domain experience -- Christian Pickett built billing at Vercel and payments at Coinbase, Bera Sogut built Maps APIs and reCAPTCHA at Google. If you believe this category will be won by developer experience rather than protocol dominance, there's a case that a focused startup moves faster than Stripe's enterprise machinery.

But that bull case has a structural flaw I can't ignore. The core product -- API aggregation, unified billing, key management -- is described even in sympathetic terms as "technically replicable." The MCP integration pattern is open-source and standardized. The 16 API partners are a start, but marketplace network effects at this scale are fragile -- any competitor or incumbent can sign the same partners. The company hasn't articulated a technology breakthrough that creates economic impact; it's articulated a better developer workflow. When I lost $160 million on KiOR, the lesson was that technology achievement without sustainable unit economics is a trap. Here the lesson is adjacent: operational convenience without structural defensibility is a feature, not a company. Stripe can replicate this product surface in a quarter and bundle it into their existing developer relationships.

The founders are competent -- Pickett's Coinbase and Vercel experience maps directly to the product, and Sogut's ICPC World Finals performance (24th place) signals raw intellectual horsepower. But this is exactly the pattern I worry about: experienced industry insiders executing the industry's playbook. They came from payments and API infrastructure, and they're building payments and API infrastructure. Where is the contrarian bet? Where is the thing that makes experts say "that's crazy"? Nobody thinks unified API access for agents is crazy -- Stripe, Nevermined, Nightmarket, and Prava all agree it's a good idea. When everyone agrees the opportunity exists, the consequence of winning a commodity race is a modest business, not a transformative one.

The timing dimension is the company's strongest attribute. They are genuinely early to a real wave -- MCP launched months ago, agentic workflows are accelerating, and the developer pain point will intensify. But being early to a wave that Stripe is also riding with 1,000x the resources is not the same as being early to a wave that incumbents have dismissed. I invest in companies where the rate of change in the underlying technology creates an opening that incumbents structurally cannot exploit. Here, the rate of change in agentic AI benefits Stripe's position as much as Orthogonal's.
