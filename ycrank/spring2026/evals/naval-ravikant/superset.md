# Superset -- Naval Ravikant Evaluation

The first thing I notice is what Anthropic is doing. They've already shipped native multi-agent orchestration for Claude Code. That's not a hypothetical competitive threat -- it's a platform owner building the feature you're selling, on the timeline that matters. When I invested in Alchemy, Ethereum wasn't trying to build a developer platform. When I invested in AngelList, Goldman Sachs wasn't trying to build angel syndicate tooling. The structural condition for a picks-and-shovels play is that the mine operators need shovels but don't want to make them. Here, the mine operators are actively making shovels. Cursor, Windsurf, and the agent providers themselves have every incentive and capability to add multi-agent orchestration natively. The question isn't whether this is a real pain point -- it obviously is -- but whether the orchestration layer is infrastructure or a feature. I see a feature.

On specific knowledge: three ex-YC CTOs who can ship fast. Kiet built Onlook to 25k GitHub stars, which demonstrates real open-source distribution instinct. That's a skill. But the insight powering Superset -- "developers running multiple CLI agents need isolated Git worktrees and a dashboard" -- is not the kind of knowledge that takes years to accumulate. Any developer who spent two weeks using Claude Code concurrently across tasks would identify this gap. Compare that to Amjad Masad at Replit, who spent years understanding the barriers between wanting to code and actually coding, or to the Notion founders who had a philosophical thesis about the relationship between tools and cognition. The Superset founders have execution knowledge -- how to ship Electron apps, how to grow GitHub stars -- but execution knowledge is transferable. I also note that two of the three founders were in YC W25, meaning they left their previous companies within months. That pattern -- serial founding with fast pivots -- can indicate either exceptional opportunity recognition or the thing I warn against most: being more passionate about founding than about the problem.

The leverage test doesn't pass cleanly. Software is leveraged by default, so there's a baseline. But Superset doesn't create a new form of leverage that didn't exist before. The leverage in AI coding is in the agents themselves -- Claude Code, Codex, Gemini. Superset makes the existing leverage marginally more convenient to deploy in parallel. That's valuable UX work, but it's not Notion giving one person the output of a team, or Uber turning every car into a taxi. It's tmux with better design. The agent-agnostic positioning is smart -- you don't compete with Cursor by building another AI IDE, you sit above the agents. But sitting above the agents means you're a thin layer that the platforms below you can absorb. The Elastic License 2.0 tells me the founders feel this vulnerability -- if they had deep technical moats, they wouldn't need legal protection against competitive hosting.

The bull case is real and I want to be honest about it. If multi-agent workflows become the dominant paradigm for professional software development -- which they might -- the company that owns the orchestration UX captures meaningful value. Eight thousand GitHub stars in five months is not noise. The structural argument that Cursor and Windsurf won't build an agent-agnostic orchestrator because it would commoditize their proprietary agents has merit -- it's a classic disruption gap. If Superset threads the needle between the AI IDE vendors and the CLI agent providers, and if they build workflow lock-in through saved configurations, team templates, and enterprise features fast enough, they could establish the standard. The founders have proven they can ship and distribute developer tools. That's the bet.

But I keep coming back to the leverage ratio. What's the output per unit of input that Superset creates that nothing else can? The underlying agents do the leveraged work. Superset manages processes. Git worktrees are a built-in Git feature. The diff viewer is nice but VS Code has one. Port forwarding exists. The aggregation of these features into one desktop app is genuinely useful but not defensible over a two-year horizon against any well-funded IDE or the agent providers themselves. This is a timing trade, not a compounding asset. I don't see the technology getting harder to replicate with each user or each iteration -- there's no knowledge graph accumulating, no network effect strengthening, no data flywheel spinning. Stack Overflow got more valuable with every question. Notion got stickier with every template. Superset processes agents in parallel, and next month's version does the same thing slightly better. The market is consensus -- AI coding tools are the hottest category since crypto in 2021. Everyone agrees this is the future. When everyone agrees, the pricing already reflects the optimism.

Pass. Strong team, real pain point, wrong leverage architecture for my portfolio. I invest in platforms that create new forms of leverage, not applications that make existing leverage more convenient. If the founders develop a deeper thesis about multi-agent coordination -- something about how agents should communicate, share context, learn from each other's work -- that would be a fundamentally different company and a fundamentally different conversation.
