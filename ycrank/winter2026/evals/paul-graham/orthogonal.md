# Orthogonal -- Paul Graham Evaluation

The first thing I notice is Stripe. Not as a competitor in the abstract, but as a concrete, present threat with 50 million x402 transactions already processed and a partnership with OpenAI on the Agentic Commerce Protocol. When I funded the Collisons, there was no Stripe — the opportunity existed precisely because nobody with Stripe's resources wanted to deal with banks and compliance. Orthogonal is building in a space where the most formidable payments company on earth has already shown up with a named product. That changes the calculus entirely. The question isn't whether agentic API payments is a real category — Stripe's presence confirms it is — but whether there's a specific wedge that Stripe structurally can't occupy. Orthogonal's answer is that Stripe went blockchain-native with USDC on Base, leaving room for a simpler, fiat-denominated, developer-friendly alternative. That's a reasonable thesis. But it's a thin structural barrier, because Stripe could extend to off-chain billing in a quarter if they wanted to.

The founder backgrounds are the strongest signal here. Pickett did billing at Vercel and payments at Coinbase — he literally built the infrastructure this product replaces. Sogut built APIs at Google, including reCAPTCHA and Maps, and is a two-time ICPC World Finals competitor (24th place globally). These are technical hackers who have worked on both sides of the problem: payments and API infrastructure. They met at McGill and have been close friends since, which clears the co-founder relationship bar. The domain knowledge is genuine and deep. But here's where I push: did they discover this problem while building agent systems themselves, or did they look at the AI agent trend and realize their combined experience positioned them well for it? The dossier doesn't mention them ever building an AI agent that needed to call dozens of APIs. Their experience is in the underlying infrastructure, not in the pain of being an agent developer. That's the difference between Drew Houston forgetting his USB drive and someone who worked at Seagate deciding to build file sync.

The schlep test is where this gets interesting — and where I'm less convinced than I'd like to be. What's the actual schlep here? API aggregation, unified billing, key management. These are real engineering problems, but they're not the kind of ugly, regulatory, soul-crushing work that creates schlep blindness. When the Collisons walked into banks and dealt with fraud compliance and money transmission licenses, that was schlep that made thousands of programmers unconsciously look away. Aggregating API access and handling billing is more like a well-scoped infrastructure product than a schlep that terrifies people. The two-sided marketplace cold start is hard, but cold starts are just marketplace mechanics — they don't deter people the way dealing with HIPAA or banking regulators does. The YC network helps with initial supply (several YC companies are listed as partners), but network-bootstrapped supply isn't the same as organic demand pulling the product into existence.

The bull case deserves real consideration. If agentic AI goes where McKinsey and everyone else projects — trillions in agent-mediated commerce — then whoever becomes the default access and payment layer for API calls captures enormous value. Stripe's blockchain-first approach may genuinely alienate the majority of developers who want simple, fiat-denominated billing. These founders have exactly the credentials to build this. Sixteen API partners already listed, integration with Claude Code and Cursor, the YC network accelerating supply-side adoption — all of this could compound quickly if agent usage crosses a threshold. The analogy would be something like Twilio in 2008: obvious in retrospect, but at the time most people couldn't see why developers needed a simple API for phone calls. If Orthogonal becomes the Twilio for agent-to-API access, the outcome is enormous.

But what holds me back is the absence of any signal that developers are actually pulling this product. No revenue. No user counts. No waitlist. Ten GitHub stars across three repos. Sixteen API partners is supply-side — it tells me the founders can do BD, not that developers need this product today. At pre-seed I don't demand traction, but I look for any organic signal: a Hacker News post that blew up, a prototype that developers found and started using without being asked, a single paying customer. I don't see that here. The idea is logical and the founders are credentialed, but logic and credentials produce what I'd call a "made up" idea — not in the sense that it's wrong, but in the sense that it emerged from analyzing a market opportunity rather than from the founders' own frustration building agents. The category is also getting crowded: Stripe, Nevermined ($7M raised), Nightmarket, Prava, and surely more coming. When multiple well-funded players converge on a category simultaneously, the startup advantage of being small and focused gets diluted.

These are competent, technical founders with genuine domain expertise attacking a real emerging category. The problem is that it's an emerging category everyone can see, the schlep isn't deep enough to deter serious competition, and I don't yet see evidence that the product is being pulled rather than pushed. I'd want to see a few hundred developers actively routing API calls through Orthogonal before I'd get excited. The founders have the skills to build this — the question is whether the market will let them own it given who else is building here.
