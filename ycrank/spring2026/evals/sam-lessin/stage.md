# Stage -- Sam Lessin Evaluation

Here's what jumps out at me immediately: Stage is tagged "SaaS" and "Developer Tools" on its own YC page. I have spent the last two years writing essays, going on CNBC, and building a 104-page deck arguing that SaaS moats are dissolving and that software is a business tool, not a business model. When someone sends me a company that self-identifies as SaaS developer tooling and asks me to invest my own money, I need to see something structurally extraordinary to override that prior. I don't see it here.

The sane-person-insane-idea calibration is off in the way that concerns me most. Charles and Dean are clearly competent -- Stanford CS, Five Rings quant trading, Dean's AI master's degree and Apple rotations. The "sane" box is checked. But the idea -- that code review UX needs to be better because AI is generating more code than humans can parse -- is not insane. It's one of the most consensus observations in developer tooling right now. YC has funded code review companies in three consecutive batches: Haystack in S24, Cubic in S25, and now Stage in Spring 2026. CodeRabbit has $88M in funding and $15M+ ARR. Greptile raised $29M. Graphite sold for north of $290M. When the entire market agrees the problem exists and is pouring hundreds of millions into various solutions, you're not buying a non-consensus bet -- you're buying a consensus bet with a slightly different UX metaphor. "Structured chapters" versus "infinite canvas" versus "stacked PRs" versus "AI bot comments" is a product design debate, not a structural thesis divergence.

The AI kill test is where this really falls apart for me, and it's the thing I think most investors evaluating Stage will miss. Stage exists because of a transitional problem: AI writes code faster than humans can review it. But the obvious solution to "AI generates code humans don't understand" is not a better diff viewer -- it's AI that explains its own code, or AI that reviews code autonomously. Anthropic launched a multi-agent code review tool for Claude Enterprise customers literally this month. The dossier notes that CodeRabbit and Greptile are already automating the review function itself. Stage is building a human-comprehension layer in the exact moment when the trajectory points toward humans being removed from the review loop entirely, or at minimum toward AI doing the comprehension work for them. This isn't "cherry on top" -- where AI makes an already-good business better. This is a business whose raison d'être is a gap that AI itself is closing from both sides.

Let me steelman this, because I take the bull case seriously even when I'm passing. What would have to be true for Stage to be a great investment? You'd need to believe that human code review is a permanent, irreducible part of software development -- that regulations, liability concerns, or fundamental software reliability requirements will always demand a human in the loop understanding what changed and why. If that's true, and if "structured chapters" becomes the standard metaphor for how humans process changesets (the way Git became the standard for version control), Stage could establish a new primitive for code comprehension. The Graphite acquisition by Cursor genuinely did remove the closest independent workflow competitor, creating a window. And the founders' shared experience at Five Rings -- where Dean built an in-house coding agent and presumably experienced the review burden firsthand -- gives them authentic problem exposure. If the chapter format achieves the kind of workflow lock-in that Graphite's stacked PRs created for its 500+ customer teams, there's a path. But that's a lot of "ifs" stacked on top of a UX innovation with no technical moat, no network effects, no physical-world operations, and no protocol-layer dynamics. The dossier itself states plainly: "any well-resourced competitor (Cursor/Graphite, GitHub, GitLab) could implement a similar organizational layer."

The structural moat question is what killed my Birchbox investment, and the pattern here rhymes uncomfortably. A UX innovation without compounding defensibility in a market where well-funded incumbents have both the incentive and the capability to replicate -- that's a company that might gain traction but can't hold territory. GitHub could add chapter-style diff organization as a native feature. Cursor, which now owns Graphite's team and technology, could build it. Any LLM integration could restructure a diff into logical sections as a prompt away. The "structured chapters" concept is clever product thinking, but it's a feature waiting to be absorbed, not infrastructure that others build on. There is no protocol here, no new primitive, no physical operation that software amplifies but cannot replace. It's software all the way down, which is exactly the category where moats are evaporating fastest.

I respect the founders' engineering chops, and the timing observation about AI-generated code overwhelming reviewers is correct. But correct observations about market pain don't make defensible businesses. Every VC in the ecosystem sees this pain point. The signal I need -- a structural reason why Stage specifically captures and retains value in a way competitors cannot replicate -- is absent. Pass.
