# Limrun -- Vinod Khosla Evaluation

Limrun is a competent pick-and-shovel for the agent economy whose strongest dimension -- the Apple Silicon fleet ops surface that Muvaffak Onuş is building -- is also the dimension on which my framework rewards least, and whose weakest dimension is the one I read first. When I map the success-state of Limrun at maximum ambition, what I get is "cloud coding agents can finally compile iOS apps." That is a real engineering wedge. It is not a civilizational variable. No incumbent regime collapses; Apple still owns the toolchain, MacStadium still owns the racks, the labor displacement happens one layer up at Replit or Cognition. Limrun is infrastructure beneath the consequence, not the consequence.

Consequence-first: the high rung on this ladder is **OpenAI** (2019), where the success-state was a civilizational rewrite of the "$15 trillion of U.S. GDP is labor" frame and that is what justified the $50M first VC check before the moat was legible. Limrun shares OpenAI's adjacency -- it serves the agents that do the displacement -- but the dossier never names the incumbent regime that Limrun itself collapses. "XCode in the cloud for agents" is a service tier, not a category death; the success-state of Limrun is that Replit's agent ships an iOS build, not that the App Store economy reorganizes. The Coconote (Quizlet) and SorceJobs (YC W25) endpoints in the dossier are workflow conveniences, not the McKinsey-1986-cellphone-forecast inversion I look for. Limrun is two rungs below OpenAI on consequence and the dossier offers no language to climb.

Technical and capital risk is where Limrun looks least like a wrapper and most like Onuş's Crossplane lineage -- the eleven-thousand-star CNCF project where he learned to virtualize cloud APIs behind controllers. The pattern of "cloudify a system that resists cloudification" is real, and Apple's macOS SLA restricting virtualization is the kind of clause that creates an actual operating moat over 18 months: every quarter Limrun runs Apple Silicon at agent-loop latency is a quarter MacStadium ($68.1M raised) does not, and every integration with Maestro, Detox, Appium compounds into a default API surface that hyperscalers won't bother to replicate until the category is obvious. But this compounds slowly and it compounds against incumbents who already own the hardware -- AWS EC2 Mac and XCode Cloud can ship an agent endpoint in a quarter once they decide to. The miracle count here is zero. This is operational excellence, not physics.

Onuş's profile is the **Glean** exception in miniature without the capital scale that bought the exception. Arvind Jain was ex-Google search building enterprise search -- pure domain continuity, and I funded it because the consequence and capital profile compensated. Onuş is ex-Upbound/SAP/QA Wolf building cloud infrastructure abstraction -- the same continuity pattern, the same "extend the playbook I already ran" shape my TED Talk warned about when I cited the UC Berkeley 28,000-prediction study where experts beat chance only slightly. Limrun's founder has not contradicted his field; he is executing the most legible adjacent move from his prior role as Crossplane maintainer. That is competence, not the instigator pattern.

Learning rate: the dossier on Limrun gives me no pivot, no public thesis update, no "we were wrong about X" moment from Onuş -- which is fair for a Spring 2026 solo-founder pre-seed, so I apply the absent-evidence default rather than penalize. The **Slash** (2024) Kanye-pivot pattern that justified my $100M Series C check is the bar; Limrun has not yet had the chance to clear or fail it. I do not score this aggressively in either direction.

Required-capital profile is the dimension where Limrun is most clearly mis-sized for my fund. The roadmap as disclosed -- streamed-minutes consumption pricing, seven agent-platform integrations, no plant, no model training, no fleet beyond what MacStadium already operates -- is fundable by any Tier-1 seed shop at $3-5M. I am over-resourced. This is not the **Commonwealth Fusion** (2019) $115M Series A shape where consensus VCs declined on capital depth; it is the shape where consensus VCs would compete to lead.

Thirty-six month forecast on Limrun: acquired by a coding-agent platform (most likely Replit or a Cognition-adjacent buyer) for $40-120M between 2027 and 2028, or absorbed as a feature when XCode Cloud opens an agent API. Limrun does not reach $50M ARR as an independent company, and Onuş ends up running mobile-build infrastructure inside an acquirer. The mechanism is the signature itself: high operational competence on a low-consequence wedge resolves through acquisition, not category creation.

The bull case that stings: Limrun's seven embedded platform integrations -- Replit, Rork, Momentic AI, Minitap AI, Droidrun Cloud, Vibecode, x1.new -- could become the default API the way Stripe became the default checkout, and if mobile agent traffic 10x's by 2027, Limrun is the toll. Jeff An at Momentic calling Limrun "world class infrastructure operators" is the kind of pre-seed customer love I usually treat as signal. But this is the **DoorDash** (2013) exception template without DoorDash's consumer-network compounding -- Limrun's customers are seven startups that can each rebuild this internally once the API surface stabilizes, and Apple's macOS SLA cuts both ways: it raises the floor for new entrants and caps the ceiling for Limrun by leaving the toolchain in Apple's hands. The condition that would flip me: a signed multi-year integration with a top-three coding-agent platform (Cursor, Cognition, or Replit at exclusive terms) plus a credible pivot in Limrun's roadmap toward on-device agent control or simulator-based RL data for mobile foundation models -- a stack-deep move that earns the **Physical Intelligence** (2024) comparison instead of the MacStadium one. Absent that, I pass.
