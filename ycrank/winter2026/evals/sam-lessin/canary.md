# Canary -- Sam Lessin Evaluation

Here's what strikes me immediately about Canary: this is an AI product built to fix problems created by AI products. The founders left Windsurf -- one of the AI coding assistants that's accelerating code generation -- to build an AI tool that tests all the code those assistants are generating. It's turtles all the way down. You've got an LLM reading code that was partly written by an LLM, generating tests that depend on the LLM correctly inferring developer intent, and the whole thing breaks if the foundation model upstream changes its pricing, performance, or behavior. The critics on the YC LinkedIn post nailed it: you're "layering a second stochastic engine" on top of a probabilistic system. This is not a business I can underwrite. It's a feature waiting to be absorbed.

Let me run my sane-person-insane-idea filter. The sane part is real but modest: Aakash Mahalingam built coding agents at Windsurf, spent time at CZI, AWS, and Morgan Stanley. The team collectively touched Windsurf, Cognition, and Google. They understand how AI coding tools interact with codebases. That's genuine domain credibility. But here's where the framework breaks down completely -- the idea is not insane. It's the most obvious idea in developer tools right now. Five competitors have collectively raised over $180 million in the last eighteen months. Momentic already has 2,600 users including Notion and Webflow. Functionize has GE Healthcare and ServiceNow. When I backed Anatoly at Solana during crypto winter, building a new Layer 1 blockchain was genuinely non-consensus -- most of the industry thought additional blockchains were unnecessary. "AI-generated code needs better testing" is something every partner at every firm on Sand Hill Road is nodding along to. That's not an insane idea. That's a consensus trade at a crowded table.

The structural moat question is where this really falls apart through my lens. I've spent the last two years arguing that software companies that had moats are discovering they don't have moats anymore. Canary is a pure software company whose entire moat IS AI capability. Remove the LLM and there is no product -- not a worse product, no product at all. This is the precise opposite of my "cherry on top" test. When I look at a company like Craftwork, the painting business works without software; the tech layer makes it better. When I look at Canary, there's nothing underneath the AI. Worse, the moat is being attacked from above: OpenAI and Anthropic are publishing agentic testing tutorials. Cursor and Windsurf could integrate test generation directly into their coding workflow -- why would a developer use a separate tool when their AI coding assistant can also write the tests? The founders left Windsurf to build this. Windsurf could ship this as a feature.

The bull case, which I want to take seriously: the code-reading approach is technically differentiated. Most competitors work at the UI/DOM layer, generating tests by observing application behavior. Canary reads source code and PR diffs to understand developer intent. If this approach genuinely produces more stable tests, and if processing thousands of codebases creates a compounding data advantage in code-to-test mappings, you could argue there's a flywheel forming. The market timing is legitimately good -- AI coding tools have exploded and QA hasn't kept pace. And the switching costs from accumulated regression suites are real, if small. For this to be a great investment, you'd need to believe that (a) the code-reading approach is defensible enough that well-funded competitors can't replicate it within a year, (b) foundation model providers won't absorb the testing use case, and (c) the AI coding tools themselves won't build testing natively. I don't believe any of those three things, but I'll acknowledge the code-reading insight is a smart technical bet even if I don't think it's a durable structural advantage.

There's no new primitive or protocol here. Canary is an application sitting on GitHub's platform, consuming LLM APIs, running Playwright browsers. One team using Canary doesn't make it meaningfully better for another team. Compare this to Helium, where every hotspot deployed made the network more valuable for every other participant, or to Venmo, where every user joining made the payment network more useful. Canary's "data moat" from processing codebases is the kind of moat that sounds good in a pitch but dissolves when you realize any competitor with equivalent LLM access and similar customer count has equivalent data. The narrative can stretch -- from QA to software quality infrastructure to software correctness -- but it's competing with five other companies telling similar stories to the same buyers. The revenue cushion is nonexistent: no pricing page, no named customers, no public metrics. This is entirely narrative-dependent at a stage where the narrative is consensus.

I pass. This is a competent team chasing the most obvious opportunity in developer tools at the exact moment when every well-funded testing company and every foundation model provider is converging on the same space. The sane-person half of my filter gets a check mark; the insane-idea half gets nothing. And the AI kill test produces the worst possible answer: AI doesn't just fail to strengthen this business -- AI IS this business, and the same AI advances that make Canary possible make Canary replaceable.
