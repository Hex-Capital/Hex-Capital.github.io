# StableBrowse -- Sam Lessin Evaluation

$96 million. That's the aggregate funding already deployed to StableBrowse's direct competitors — Browserbase at $40M and a $300M valuation, Browser Use at $17M with 50K GitHub stars, Steel.dev at $17M, Kernel at $22M — all targeting the exact same buyer: developers building AI agents that need to interact with the web. When I walk into a category and see that kind of capital concentration from Felicis, Accel, and every other tier-one firm, I know exactly what I'm looking at: the upside is in the price. The market has consensus that AI agent browser infrastructure is a real category. Nobody is sleeping on this. The question for StableBrowse isn't "is this a real market?" — obviously it is. The question is whether StableBrowse is buying into a market where the risk-reward is already fully reflected, and the answer is unambiguously yes.

StableBrowse's strongest counterargument is architectural: stripping the rendering engine entirely to produce a "machine-native web engine" is a genuinely different bet than what Browserbase or Browser Use are doing. 130.6 MB per instance versus Chromium's 500MB+, 4ms cold start, 85% token reduction — if those numbers hold at scale, that's a fundamentally cheaper primitive. An advocate would say this is analogous to Solana's architectural bet against Ethereum: everyone was wrapping or augmenting Chromium the way everyone was building on Ethereum, and StableBrowse is proposing an entirely different execution architecture. The advocate would say the rendering-free engine IS the new primitive — that "machine-native browsing" will be recognized as a distinct category the way "proof of history" eventually distinguished Solana from the EVM consensus. I take this argument seriously. But it breaks down on two fronts. First, Solana was a protocol — infrastructure that compounds with network effects and validator lock-in. StableBrowse is an API with CDP compatibility explicitly designed for drop-in replacement, which means anything can also drop-in replace StableBrowse. The switching costs work in both directions. Second, the web compatibility question is brutal: stripping visual rendering while preserving JavaScript execution across canvas elements, WebGL, CSS-dependent layouts, and rendering-based anti-bot detection is an unproven claim. StableBrowse's addressable surface of the modern web might be materially smaller than the full web, and that gap is invisible until production.

Somansh Shah (Purdue DS '25, Lenovo data science intern) and Deepit Shah (UIUC CS BS/MS, Amazon intern) bring relevant technical grounding — ML and systems engineering respectively. But StableBrowse has no founder with a prior exit, no repeat relationship I can evaluate across multiple ventures, and frankly, two of the four founders — Sarthak Awasthi and Jay Mehta — can't even be confirmed as the right individuals from public data. Deepit Shah's LinkedIn indicated he was returning to Google full-time, which raises commitment questions I can't resolve from a dossier. The "sane person, insane idea" ratio doesn't activate here: the idea isn't insane. Building better browser infrastructure for AI agents is the most rational, consensus-validated thing a technical team could pitch in Spring 2026. When I backed Teamshares, people thought buying Main Street businesses and converting them to employee ownership was lunacy. Nobody thinks "AI agents need better browsers" is lunacy. It's on every VC's thesis deck.

StableBrowse fails the AI kill test with extreme prejudice. OpenAI is building ChatGPT Operator. Google has Gemini's browsing capabilities. Anthropic is building native tool-use browsing into Claude. These are not hypothetical threats — they are shipping products from companies with distribution to every AI agent developer on earth. StableBrowse is pure software infrastructure with no physical-world moat, no protocol-level network effects, no creator audience, no regulatory license. It's the exact category I described on CNBC: "Software companies that had moats are learning they aren't moats anymore." When Metropolis seeds AI-powered parking, nobody at Google can replicate the parking infrastructure. When StableBrowse builds a rendering-free browser API, every LLM provider's infra team can build the equivalent as a feature. CDP compatibility makes this worse, not better — StableBrowse positioned itself as a drop-in replacement, which means it's designed to be commoditized.

The one verifiable condition that would flip me: if StableBrowse demonstrated that its rendering-free architecture creates a data flywheel — proprietary semantic maps of the web's structure that compound with usage and cannot be replicated by competitors starting fresh — that would be a protocol-like moat. But there's zero evidence of this today, no GitHub repos, no traction data, no customer signals. StableBrowse enters a consensus category with standard YC funding against $96M in competitor capital, with first-time founders whose commitment levels are partially unverifiable, building pure software that LLM providers are already commoditizing. The pricing is wrong — but in the wrong direction. The market is overvaluing this category, not undervaluing it.
