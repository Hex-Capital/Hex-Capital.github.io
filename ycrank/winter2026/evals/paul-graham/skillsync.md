# Skillsync -- Paul Graham Evaluation

The first thing I notice about Skillsync is that this is not a schlep-blindness opportunity. When I look at a problem and ask "why hasn't anyone done this already?", the best answer is "because the work is so ugly that smart people unconsciously look away." That's what made Stripe work — everyone knew payments were broken, but nobody wanted to deal with banks and compliance. With Skillsync, the answer to "why hasn't anyone done this?" is "multiple companies already have." Prog.AI raised a million dollars analyzing GitHub code to infer developer skills across 60M+ profiles — and they got TechCrunch coverage in 2023. SeekOut raised $186M. hireEZ raised $76M. The problem of finding specialized engineers is real, but it's not hiding. It's a well-funded battlefield. And the core technical capability — using LLMs to parse public GitHub code — is becoming more commoditized by the month, not less. When the schlep is primarily computational rather than regulatory or logistical, it doesn't create the structural moat that keeps competitors away.

The organic origin story is the strongest part of this company. Both founders worked together at Juspay on Hyperswitch, a Rust-based open-source payments switch they grew from zero to 20k+ GitHub stars. They lived inside the open-source ecosystem and saw firsthand that brilliant contributors were invisible to recruiters. This passes the organic test cleanly — they didn't sit down and brainstorm "what's hot in HR tech." They stumbled into this problem through their own experience growing an open-source community. That matters. But organic discovery is necessary, not sufficient. Drew Houston organically discovered file sync was broken, but the reason Dropbox worked was that the solution required deep systems engineering that most people couldn't replicate. What's the equivalent here? A "developer knowledge graph" built from LLM-based code analysis of public repositories. Any team with GPT-4 access and the GitHub API can attempt this tomorrow.

The pivot from internal tool to external recruiting platform within six months is worth examining. In June 2025, the Product Hunt launch was "Discover hidden talent in your codebase" — an internal skill-mapping tool. By late 2025 they'd repositioned as "Find anyone in open source" — an external sourcing platform. I don't penalize pivots per se. Justin.tv pivoted to Twitch. But those founders had years of data telling them game streaming was where the pull was. Skillsync pivoted in under six months with 3 Product Hunt upvotes on the original product. That reads less like "we found where the real demand is" and more like "we're still searching." The current positioning may not be final either.

The bull case deserves a full paragraph. Nishant Joshi is a genuine hacker — a top-percentile Rust developer with 148 GitHub followers and serious systems programming chops. That's rare and valuable. Most recruiting tools are built by recruiting people who don't deeply understand code quality. If Skillsync's code analysis is meaningfully better than Prog.AI's because the founders actually know what excellent engineering contributions look like from the inside, the "Cursor-like" search experience could create real product differentiation. Search is a domain where quality differences compound — Google beat a dozen search engines not through more data but through better ranking. If Skillsync's rankings are noticeably more accurate for specialized roles (WASM compilers, edge ML, low-latency trading), word of mouth among engineering leaders could create organic pull that broad-spectrum tools can't match. The fact that both founders share deep context from working together on the same open-source project means they have real co-founder alignment, not a partnership of convenience.

But even the bull case has limits. The website claims "hundreds of talent teams" using the platform, but there's no verification — no named customers, no case studies, no organic testimonials. Three Product Hunt upvotes is essentially zero signal. At $499/month the unit economics could work if the product delivers, but I see no evidence yet that anyone would be genuinely harmed if Skillsync disappeared tomorrow. The addressable pool is also structurally limited — only engineers who contribute to public open-source repos on GitHub are discoverable, which excludes the majority of skilled engineers working on proprietary codebases. That ceiling matters when you're competing against platforms that surface candidates from all sources.

This is a competent team working on a real problem they discovered organically. What's missing is the structural advantage. When I funded Stripe, the advantage wasn't just that the Collisons understood payments — it was that the schlep of dealing with banks, fraud regulation, and PCI compliance created a moat that kept fast followers out for years. Skillsync's code-analysis capability, built on publicly accessible data using increasingly available LLM tools, doesn't have that quality. The founders are strong enough that I'd want to see where they end up in a year. But with my own money, I need to see either a deeper technical moat or undeniable user pull before writing a check.
