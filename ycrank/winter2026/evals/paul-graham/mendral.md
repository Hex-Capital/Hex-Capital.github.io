# Mendral -- Paul Graham Evaluation

Sam Alba was Docker's first hire. Andrea Luzzardi built Docker Swarm. Together they spent a decade inside the guts of CI/CD infrastructure, then co-founded Dagger and took it through YC in Winter 2019, raising $30.4M to build a programmable CI/CD engine. Now they've left that company to build Mendral. When founders who've already raised significant money walk away to start something new, there are two possible explanations: either they got pushed out, or they saw something they couldn't ignore. The dossier doesn't tell me which. But the product they're building -- an AI agent that autonomously investigates CI failures, traces root causes across billions of log lines, and opens fix PRs -- is so precisely fitted to what these two specific humans have been doing for fifteen years that the origin story writes itself. They didn't brainstorm this idea. They've been watching engineers suffer through broken builds since 2010. The LLM capability threshold finally crossed, and they knew exactly what to build because they'd been staring at the problem the entire time.

The schlep here is what makes me pay attention. Every developer hates debugging CI failures. It's the janitor work of software engineering -- scanning thousands of log lines, context-switching from feature work, manually tracing flaky tests through build configurations. Thousands of smart engineers have experienced this pain. But building the solution requires processing billions of log lines into a queryable store, running sandboxed investigations inside Firecracker microVMs that boot in under 125ms, achieving SOC 2 Type II compliance before you have meaningful revenue, and -- the hardest part -- earning enough trust that engineering teams let your AI agent open pull requests in their production codebase. That last piece is the real schlep. Code suggestions are easy to sell. Autonomous code changes? That's a trust surface area that scares everyone, including the incumbents. GitHub could theoretically extend Copilot into CI remediation, but autonomously writing code into customer repos is a fundamentally different liability model than offering suggestions in an IDE. The incentive misalignment is structural: CI platforms sell pipeline minutes, not autonomous code changes. This is exactly the kind of problem that hides behind schlep blindness -- obvious to everyone, tackled by almost no one, because the work is ugly and the trust barrier is enormous.

These are not ordinary pre-seed founders. They are among the most deeply qualified people on Earth to build this specific product. Sam scaled Docker's engineering from 3 to 100+ engineers, overseeing Docker Desktop, Compose, and the CI-adjacent tooling that millions of developers use. Andrea built distributed systems at Google's Storage Infrastructure group before becoming Docker's lead architect. Both write code -- Sam's GitHub has a Docker client library in Go, Andrea's has a 1.4K-star SSH bridge. The custom agent loop is written in Go with a tiered LLM routing strategy (Haiku for parsing, Sonnet for deduplication, Opus for root-cause analysis) that reflects genuine architectural thinking about cost and accuracy tradeoffs, not just "throw everything at GPT-4." More importantly, they have a fifteen-year working relationship spanning EPITECH, Docker, and Dagger. Co-founder durability like that is rare and invaluable. This is not a team that met at a hackathon.

The traction signals are meaningful. Five paying customers including PostHog -- a real company, not a friend's side project. Processing 575,000 CI jobs per week from PostHog alone, covering 1.18 billion log lines. Closing 16,000 CI investigations autonomously per month. When a customer is routing half a million CI jobs per week through your system, that's not a trial. That's dependency. The customer list -- PostHog, Inngest, Luminai -- suggests founder-led sales leveraging their Docker and Dagger network, which is exactly what doing things that don't scale looks like at this stage. They're not running paid acquisition campaigns; they're calling people they know in DevOps and saying "let us fix your CI."

The Dagger question is the one thing that gives me pause. They co-founded Dagger with Solomon Hykes, raised $30.4M, and Dagger is still an active company. The dossier has no public information about why they left or whether there are IP or non-compete complications. This rhymes with the Justin.tv pattern -- founders who went through the crucible once and came back with sharper insight -- but without knowing the Dagger circumstances, I can't fully resolve the risk. If they left because they realized autonomous CI remediation was a bigger opportunity than programmable pipelines, that's a strong signal of taste and conviction. If there was a messy split, that creates overhead I don't want in a portfolio company.

The competitive landscape is less threatening than it looks on paper. Trunk.io detects flaky tests and quarantines them -- that's the diagnostic half without the fix. Harness is a sprawling platform that won't narrow its focus to autonomous code remediation. The real threat is GitHub extending Copilot into CI, but as I said, the trust model for autonomous code changes is categorically different from code suggestions. Mendral's accumulating dataset of failure patterns across customer codebases creates compounding value that gets harder to replicate over time. The 35:1 log compression ratio and the Firecracker sandboxing represent genuine engineering depth, not API wrappers. I'd invest. The organic problem-founder fit is as strong as I've seen at pre-seed, the schlep is real and structurally deters competition, and the founders have already demonstrated they can build and ship. The Dagger transition is a question mark, not a red flag.
