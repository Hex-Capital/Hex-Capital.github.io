# Replicas -- Vinod Khosla Evaluation

Let me be direct about what I see here: Replicas is building plumbing for other people's revolution. The actual disruption in software engineering is happening inside Claude Code and Codex -- the agents that can reason about code, fix bugs, implement features. Those are the technologies transforming a multi-trillion-dollar industry. Replicas is a convenience layer that spins up VMs and dispatches tasks from Slack. If Replicas achieves total market dominance, the consequence is that developers click a button in Slack instead of opening a terminal. That is not the kind of outcome I get out of bed for. The consequence of success here is a useful infrastructure business, not an industry transformation. When I backed Square, mobile payments didn't just make existing POS systems slightly more convenient -- they made the entire merchant acquiring infrastructure structurally obsolete. Replicas doesn't make anything obsolete. It makes an already-powerful tool marginally easier to invoke.

The commoditization dynamics are brutal and structural, not speculative. OpenAI's Codex already runs in cloud sandboxes natively -- that is the agent provider building the exact execution layer Replicas sells. Anthropic is developing Claude Code Channels. The dossier itself acknowledges that "the core functionality -- spinning up a VM, cloning a repo, running a CLI agent -- is technically reproducible," and that open-source alternatives already exist. The "agent-agnostic" positioning assumes the major providers won't vertically integrate, but they will, because cloud execution of their own agent is a natural extension of their product. This is the infrastructure equivalent of building a better phone case while the phone manufacturer is about to include one in the box. Devin has $900 million and $150M ARR. Factory has $50 million. Augment has $252 million. Replicas is a solo founder with a V1 against nearly $1.5 billion in aggregate competitive capital, and the incumbents aren't even the biggest threat -- the agent providers are.

The strongest bull case requires you to believe that coding agent fragmentation persists indefinitely and that the orchestration layer captures durable value -- essentially, that Replicas becomes the Switzerland of agent dispatch, the way Twilio became the neutral API for communications. If three to five agents remain competitive and engineering teams want to run different agents for different task types, a well-integrated neutral platform could become genuinely sticky. Connor Loi's Helicone background is relevant here -- he saw LLM API traffic at 2.1 billion+ requests, he built the AI Gateway for request routing, and he identified the gap between agent capability and agent deployment convenience. That is a real insight born from proximity. The timing is also genuinely favorable: the shift from synchronous copilot assistance to asynchronous agent execution is underway and accelerating. But even in the best case, the Twilio analogy breaks down because Twilio had deep carrier integration complexity as its moat. Replicas' moat is Slack and Linear integrations -- those are API calls, not infrastructure barriers.

Connor is clearly a capable builder -- shipping across Helicone's core platform for eight months, winning hackathons, producing a breadth of side projects from ray tracers to space RTS games. But nothing in the dossier suggests contrarian courage or the willingness to attempt what experts consider impossible. This is a consensus bet: everyone agrees AI coding agents need better infrastructure. He is an insider from the LLM tooling ecosystem building the adjacent obvious product, not an outsider attacking a problem the industry dismissed. I look for founders who make people look at them funny. Nobody looks funny at someone building cloud infrastructure for AI agents in 2026 -- that is the most predictable startup thesis in San Francisco right now. And as a solo student founder with no hiring plans, the operational constraint is real: product, engineering, sales, support, and fundraising rest on one person against teams with hundreds of engineers and billions in capital.

I have enormous respect for the trend Replicas is riding. Asynchronous AI coding agents are genuinely reshaping how software gets built, and the trajectory of agent capability improvement is exponential. But the right question is not whether this trend is real -- it is -- but whether the orchestration layer is where the value accrues. History suggests the value concentrates in the agent models themselves and in the developer platforms that control distribution (GitHub, IDE ecosystems), not in thin middleware that routes tasks between them. You are too smart to spend your career as the plumbing between other people's breakthroughs. Go build the agent itself, or go find a problem where the consequence of success terrifies and exhilarates you at the same time.
