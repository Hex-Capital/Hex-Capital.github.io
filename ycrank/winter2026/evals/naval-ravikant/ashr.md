# Ashr -- Naval Ravikant Evaluation

The first thing I notice is the competitive landscape: Janus, Coval, Maxim AI, LangWatch, Langfuse, Braintrust — and that's just the startups. Datadog, LangChain, and eventually OpenAI and Anthropic themselves are circling this space. When five funded companies and multiple open-source projects are all building variations of "test your AI agent," you're not betting against a gloomy crowd. You're joining a crowded consensus. The whole premise — "AI agents need better testing tools" — is a PowerPoint slide that every developer tools investor has already seen twenty times. Nobody disagrees. And when nobody disagrees, the asymmetric upside has been priced away before you even write a check.

The specific knowledge question is where this falls apart for me. Two years of building AI agents in finance and DevOps is relevant experience, but it's also the resume line of several thousand engineers who graduated from Berkeley, Stanford, and CMU in the same window. What does Shreyas know about testing agent behavior that the Coval founder — who was tech lead at Waymo, arguably the most sophisticated simulation environment ever built for autonomous systems — doesn't know better? What does Rohan know that the Janus founders — Carnegie Mellon ML researchers with Microsoft eval framework experience — can't articulate more deeply? A prior exit with Ask Geri (an AI companionship app for seniors) demonstrates execution ability, which I respect, but it doesn't generate the kind of non-transferable domain insight that would make me confident these founders will outbuild better-capitalized competitors in a feature race. The pain they've experienced is real, but it's the same pain everyone in the agent ecosystem has experienced. Shared pain isn't specific knowledge.

The leverage architecture is standard SaaS — baseline code leverage, nothing more. An SDK and a dashboard. Custom evaluation metrics and business-goal alignment sound useful but also sound like each customer engagement requires configuration effort that scales semi-linearly. Compare this to Stack Overflow, where every question answered makes the platform more valuable for every future visitor without any marginal effort. Or Alchemy, where every new blockchain app built on the API strengthens the infrastructure layer. Ashr's scenario generation might accumulate useful test libraries over time, but the dossier shows no evidence of a network effect or data flywheel that would compound. The technology underneath — using LLMs to generate synthetic user journeys — is not difficult to replicate. Any competent team with API access to a frontier model can build multi-step scenario generation in a few months.

The bull case would require believing that agent testing becomes a distinct, durable category — not absorbed by observability platforms like Datadog, not bundled by LLM providers, not commoditized by open-source. You'd need to believe that multimodal support (voice, image, file generation) creates meaningful switching costs, and that the founders' execution speed lets them build enough integration surface area to lock in customers before better-funded competitors catch up. If the agent ecosystem grows at the projected 44.6% CAGR, even a mid-tier testing tool could reach meaningful scale. And Rohan's prior exit, however small, suggests he knows how to ship product and close a chapter. In a scenario where enterprise agent deployment accelerates faster than platform providers can build native testing, a standalone tool wins. But that's a narrow corridor — it requires incumbents to be slow and competitors to be slower, simultaneously.

I don't see the compounding bet here. The technology is generative scenario creation on top of LLMs — useful, but not the kind of difficult technology that gets harder to replicate over time. The founding team is technically capable but doesn't possess knowledge that would take a competitor years to accumulate. The market timing is consensus, not contrarian. The product sits at the application layer, not the infrastructure layer. This is a feature looking for a company, in a space where five other funded teams are building the same feature with comparable or better domain credentials. I'd need to see something that surprised me about their understanding of agent failure modes — a non-obvious insight about why testing frameworks built for deterministic software fundamentally can't work for stochastic agents, articulated in a way that made me update my own model. I don't see that signal here.
