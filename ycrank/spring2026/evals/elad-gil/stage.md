# Stage -- Elad Gil Evaluation

$334.5 million in combined funding across CodeRabbit, Graphite, Greptile, and Qodo -- that's the capital already deployed into AI code review before Stage wrote its first line of product code. When I evaluate market structure timing, I'm looking for a window that consensus has missed. Stage's underlying thesis -- that AI-generated code creates a comprehension gap in code review -- is correct. But this isn't a nonobvious insight in Spring 2026. CodeRabbit closed a $60M Series B in September 2025. Cursor acquired Graphite for $290M in December 2025. Qodo raised $70M Series B in March 2026. Anthropic shipped Claude Code Review as a dedicated product in March 2026. The structural shift (AI coding assistants reaching mass adoption) already happened, and the capital markets have already priced it in. Stage is arriving after the window has been recognized and occupied. Compare this to when I looked at Coinbase in 2012 -- crypto infrastructure was dismissed by every mainstream investor, and the structural shift in digital assets was invisible to consensus. Stage's timing is the inverse: the shift is real, but everyone already sees it.

Stage's "chapters" approach -- breaking pull requests into narrative sequences rather than raw diffs -- targets the presentation layer of code review. When I apply my infrastructure-layer test, Stage fails cleanly. Stage doesn't build a platform that other tools depend on. It doesn't create API-level integrations that compound switching costs. It doesn't generate a data asset that improves with scale. The dossier is explicit: "The 'chapters' concept is a presentation-layer innovation rather than a deep technical or data moat, making it reproducible by any competitor with front-end engineering resources." This is the pattern I've consistently avoided -- an application sitting on someone else's infrastructure. GitHub could add structured diff views as an optional mode tomorrow; the dossier itself concedes the structural barrier is "weak." When I backed Stripe, the thesis was that Stripe would become the payments API that entire businesses built on, creating switching costs that compound over years. Stage's "chapters" UX creates no such compounding dependency. If a team stops using Stage, they revert to GitHub's default diff view with zero data loss and zero workflow disruption.

Dean Stratakos's background building an internal coding agent at Five Rings and leading company-wide AI initiatives is the strongest signal in Stage's dossier. Combined with Charles Pan's experience as an early engineer at Yuzu Health, the founders have direct domain exposure to both AI code generation and fast-shipping team dynamics. Both are Stanford CS graduates with strong technical credentials -- Stratakos's three Apple internships and work on high-performance trading systems in C++ demonstrate genuine engineering depth. But I've seen many technically excellent founding teams get crushed by market structure. The honest assessment here is that Stage's founders are well-credentialed engineers without evidence of scaling organizations or building distribution -- and in a market this crowded, distribution capability matters as much as product quality.

The strongest bull case for Stage runs like this: CodeRabbit and Qodo add AI commentary to diffs, but Stage reorganizes how humans comprehend changes -- a fundamentally different product category. The Cursor-Graphite acquisition at $290M proves code review is a strategic acquisition target. Stage could build the best comprehension layer and get acquired by a platform player. And the problem is growing: as AI writes more code, the review bottleneck compounds. I take this seriously. But the bull case requires Stage to win a UX war against CodeRabbit ($15M ARR, 8,000+ paying customers) and Graphite (now integrated into Cursor with 500+ companies) -- companies that have already shipped structured review features and are iterating with deep engineering teams. The "chapters" concept is not defensible against organizations spending tens of millions on code review product development. Stage needs both a product miracle (chapters must be dramatically better than competitor implementations) and a distribution miracle (must acquire users against entrenched incumbents with existing integrations) -- that's two independent miracles, which my framework penalizes heavily.

The one verifiable condition that would flip me to Invest: if Stage demonstrated that its structured review approach measurably reduces review time or defect rates by 3x or more versus CodeRabbit or GitHub's native review -- validated by usage data from real engineering teams, not a demo. A 3x improvement would indicate that "chapters" isn't a UX polish but a genuine paradigm shift in how humans process code changes, creating the kind of product-market fit that survives competitive pressure. Without that evidence, Stage is a well-intentioned presentation-layer bet entering the most consensus-crowded category in developer tools. The two dimensions that lock this verdict are Nonobvious Market Timing (the AI code review window is already fully priced by capital markets) and Infrastructure-Layer Positioning (Stage builds a UI layer, not a platform layer). Both score in the bottom quartile of my framework.
