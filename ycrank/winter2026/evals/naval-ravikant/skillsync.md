# Skillsync -- Naval Ravikant Evaluation

The first thing that strikes me about Skillsync is what it isn't: it isn't infrastructure. It's an application sitting on top of someone else's platform -- GitHub, owned by Microsoft -- analyzing public data that any competitor can also access. My portfolio tilts heavily toward picks-and-shovels plays for a reason: when you own the platform layer, you own the leverage. When you're an application parsing another platform's data through their API, you're renting your existence. Microsoft could integrate GitHub contribution analysis into LinkedIn Recruiter with a single product sprint, and the entire thesis collapses. That structural dependency is the first thing I'd want the founders to convince me isn't fatal.

The specific knowledge question is where this gets interesting -- and ultimately where my conviction stalls. Both founders worked at Juspay on Hyperswitch, a Rust-based payments switch they helped grow from zero to 20,000+ GitHub stars. That's genuine open-source operating experience. They've watched developers contribute code and formed intuitions about which patterns of contribution signal real expertise versus surface-level involvement. Nishant is described as a "Top 1% Rust dev" with deep systems programming chops. This isn't nothing. But the knowledge they possess is about the *supply side* -- how developers work in open source -- not about the *demand side* -- how recruiting teams buy, adopt, and retain sourcing tools at scale. Aaditya's XLRI management and HR background fills part of that gap, but understanding organizational psychology from a classroom is different from having spent years inside recruiting operations. The observation that "GitHub contributions reveal more than resumes" is widely held by anyone who's ever hired an engineer. The question is whether their code-analysis implementation captures something genuinely non-obvious about developer capability -- and I don't see evidence of that yet.

The leverage architecture is standard vertical SaaS. One engineering team builds a product, many recruiting teams pay $499/month to use it. That's baseline software leverage -- real, but unremarkable. Where's the network effect? Developers don't sign up, so there's no supply-side flywheel. There's no marketplace dynamic, no protocol, no permissionless coordination mechanism. Each customer acquisition requires founder-led sales ("Book a Demo" / "Talk to Founders"). Compare this to Stack Overflow, which I backed: developers voluntarily contributed answers, creating a knowledge graph that grew more valuable with every interaction. That's compounding leverage. Skillsync's "developer knowledge graph" is built by scraping public repositories -- an asset any well-resourced competitor can replicate with LLMs and API access. Prog.AI already has 60M+ profiles indexed and a founder with a prior exit to Dun & Bradstreet.

The contrarian timing test fails clearly. "AI for recruiting" is one of the most consensus categories in 2026. SeekOut raised $186M. hireEZ raised $76M. Every recruiter knows GitHub profiles matter. Nobody smart is dismissing this idea -- they're just funding five other companies doing similar things. The LLM-enables-deep-code-analysis thesis is real, but it applies equally to every competitor in the space. When the timing catalyst benefits everyone, it's not a source of asymmetric advantage for any single company.

The strongest bull case: if their Hyperswitch experience translates into materially superior code analysis -- if they can parse developer capability from contribution patterns in a way that Prog.AI and others cannot -- then the "Cursor-like" search interface could create genuine differentiation in a high-willingness-to-pay market. Technical hiring managers who've used keyword-matching tools and hated them would love a natural-language interface that actually understands engineering domains. "Hundreds of talent teams" at $499/month would be meaningful revenue if verified. And the pivot from internal codebase analysis to external recruiting could represent rapid learning rather than confusion -- they found the higher-value application of their core technology. If the knowledge graph accumulates proprietary signal from search queries and hiring outcomes over time, it could develop a data moat that raw GitHub scraping can't replicate. That's the path to compounding. But it requires years of customer usage data, and they're at pre-seed with unverified traction claims and 3 Product Hunt upvotes.

I'd pass. Smart founders with relevant experience, building in a real market, but fundamentally this is an application-layer tool in a consensus category with multiple funded competitors, dependent on a platform it doesn't control, creating efficiency rather than a new category of leverage. The specific knowledge is about the data source, not the customer. The technology doesn't compound structurally. The pricing model is linear. Nothing here triggers the asymmetric upside I need to see.
