# SideKit -- Peter Thiel Evaluation

The competitive landscape here tells you everything you need to know before you evaluate anything else. SideKit is entering a market where Firebase commands 72% iOS integration and near-total Android penetration -- for free. PostHog has raised $180 million and already bundles analytics with feature flags in an open-source package. LaunchDarkly has $330 million in funding. Statsig was just acquired by OpenAI for $1.1 billion. And now a two-person team from Waterloo proposes to win this market with a 244 kilobyte SDK and $29-per-month pricing. This is not a company escaping competition. This is a company volunteering to enter a Girardian war of all against all, where multiple well-funded players are converging on the same thesis -- that developers want bundled, simpler tooling -- and imitating each other into mutual margin destruction. When I see five funded companies and a Google-backed incumbent all occupying the same conceptual space, I see the absence of a secret, not an opportunity.

What is SideKit's secret? I have looked for one and cannot find it. "Mobile developers want simpler production tools" is not a truth that most intelligent people reject -- it is a truth that Firebase, PostHog, LaunchDarkly, and Statsig have all built billion-dollar companies around. "Privacy-first analytics will win" is directionally consensus in a post-GDPR world; Plausible, Fathom, and a dozen others already pursue this thesis. "Bundling analytics with version management and feature flags is better than using three separate tools" is a packaging insight, not a secret about the world. The self-healing AI feature -- agents that process feedback and ship fixes as pull requests -- is the only element that gestures toward something genuinely new. But it sits at the highest pricing tier with zero public evidence of customer usage or effectiveness, and it is built on commodity LLM APIs that any competitor could integrate in a sprint. A feature that relies on external AI infrastructure you do not control is not proprietary technology.

The strongest bull case requires you to believe something specific: that AI-assisted code generation will create a massive new class of indie mobile developers who are structurally underserved by existing tools, and that SideKit will become to this new developer class what Stripe became to web developers -- the default infrastructure layer. This is not an unreasonable thesis. Stripe's secret was that payments were unnecessarily complex because incumbents optimized for enterprise sales, and a radically simpler API would create switching costs that compound over years. If SideKit could replicate this dynamic -- becoming so deeply embedded in indie developers' workflows that migration becomes prohibitively painful -- the small initial market could expand concentrically into larger teams and enterprise. But the analogy fractures on inspection. Stripe's switching costs are enormous because payment systems touch every transaction in a business; swapping an analytics SDK is a week of engineering work. Stripe created a category that did not exist (developer-first payments API); SideKit bundles three categories that already have dominant incumbents. And Stripe's 10x improvement was visceral -- seven lines of code to accept payments versus months of bank integration. SideKit's improvement over Firebase plus a version-check endpoint is incremental, not discontinuous.

Ashish Selvaraj's path to this problem is organically sound -- he built an iOS app, found the analytics tooling inadequate, built his own, and generalized the solution. Paul Graham would find this origin story compelling, and I understand why. The founder-market fit is real. But organic discovery of a problem is not the same as identification of a secret. Selvaraj discovered that production tooling is fragmented -- a truth every mobile developer knows. The question is whether his response to that truth creates a monopoly or merely another competitor in a crowded field. The evidence points overwhelmingly toward the latter. Seven GitHub stars, 12 commits, 227 LinkedIn followers, no public revenue, no named customers, no Android SDK -- these are not disqualifying at pre-seed, but they offer no countervailing signal against the structural competitive problem. The unverifiable team credentials (the website claims "Jane Street, Citadel" but no public record confirms Ethan Pronev's employment at either firm) add a small but real credibility concern.

The technology here is a 1-to-n improvement -- better bundling of existing capabilities -- not a 0-to-1 creation. No capability that SideKit offers was previously impossible. A smaller SDK is a marginal improvement. Privacy-first analytics is a design choice, not a technological discontinuity. Version gating is a feature that an engineer could build in a day. The only potentially discontinuous element, self-healing AI, is unproven and non-proprietary. When I invest, I need to believe that success creates a position that is structurally unassailable for a decade or more. I cannot construct that belief here. Firebase's distribution advantage, PostHog's open-source community, and the commodity nature of the underlying technology all point to a market where SideKit's advantages are temporary and replicable. This is the kind of company where the founders work hard, build a real product, acquire some customers, and ultimately discover that they are competing -- which means they are losing.
