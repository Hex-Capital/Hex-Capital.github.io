# Superset -- Paul Graham Evaluation

The first thing I notice is that Anthropic has already shipped native multi-agent orchestration for Claude Code. That's in the dossier. When the company whose CLI tool you're wrapping is building the exact feature you sell, you're not in a startup -- you're in a grace period. This is the single most important fact about Superset, and it's the one that would keep me up at night if I'd written a check.

The origin story here worries me. Three ex-YC CTOs who were building separate companies -- Onlook, Adam, Untether Labs -- left those companies and formed a new team around a problem that every developer using Claude Code can see from a mile away. The repo was created in October 2025, months after CLI agents became mainstream. When I ask "how did they find this problem?" the answer is essentially "the same way everyone else found it: by using the tools." That's not schlep blindness -- nobody is unconsciously avoiding the problem of orchestrating multiple AI agents. It's obvious. Hundreds of developers have probably thought "someone should build a better tmux wrapper for this." The reason it hasn't been built by a dominant player yet isn't that the schlep is enormous -- it's that the platforms are still deciding how much orchestration to build natively. Stripe worked because thousands of programmers avoided the banking and regulatory pain for a decade. Multi-agent orchestration has been a visible need for maybe twelve months, and the complexity involved is moderate -- Git worktree management, process supervision, a diff viewer. This is good engineering, not a painful slog that structurally deters competition.

The co-founder dynamic also triggers a pattern I've seen go wrong. The dossier says "no shared prior employer or university identified." Kiet and Avi overlapped in the W25 batch; Satya was in W23. They're all technically strong, and they've all been through YC, but they formed this team specifically to start this company. I once reported that nine companies in a batch added co-founders they didn't know well, and all nine fell apart within a year. Now, these three aren't strangers -- the YC network creates real bonds -- but it's a different thing from the Collison brothers or Huffman and Ohanian building something together out of years of shared context.

Here's the bull case, and it's not nothing. Kiet Ho built Onlook to 25,000 GitHub stars. That's a legitimate demonstration of the ability to ship open-source developer tools that developers actually adopt. And Superset itself has 8,000 stars in about five months, hit #1 on Product Hunt, and has testimonials from engineers at Microsoft, Netflix, and OpenAI. These are real signals of user pull -- you can't manufacture 595 forks. If the future of software development involves developers juggling five or six competing AI agents from different providers, then an agent-agnostic orchestration layer could be genuinely valuable, the way Docker standardized containers despite every cloud provider wanting their own. The team clearly knows how to build and ship. If I squint, I can see a world where Superset becomes the default interface for the multi-agent era -- the thing every developer opens first in the morning -- and the enterprise revenue follows naturally. For this to work, though, two things have to be true: the market has to stay fragmented (many agents, no single winner), and the orchestration layer has to develop enough depth that it can't be replicated as a feature by an IDE vendor in a quarter. I'm not convinced of either.

The commodity risk is severe and structural. Claude Squad already has 6,600 GitHub stars doing a simpler version of the same thing. Cursor is valued at $29.3 billion and has every incentive to add multi-agent orchestration if the workflow proves valuable -- and it would be a feature for them, not a company. The Elastic License 2.0 choice is telling: it's source-available but not truly open-source, which limits community contribution relative to permissively licensed alternatives. Meanwhile, the business model is a question mark -- no revenue, no pricing, a vague reference to "enterprise." At some point stars have to become dollars. The team is capable enough that they might find the path, but the path requires the upstream platforms to cooperate by not building the features themselves, and that's a bet I'm not comfortable making.

These are clearly talented hackers who can build and ship. Under different circumstances -- a harder problem, a deeper schlep, more structural defensibility -- I'd be interested. But this feels like exactly the pattern I've cautioned about: smart engineers who see a hot market and build something that fills a gap everyone can see. The gap is real. The question is whether it stays a gap, or whether it gets filled by the platforms that created it. I think it gets filled.
