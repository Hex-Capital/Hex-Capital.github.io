# Superset -- Sam Lessin Evaluation

The thing that jumped out at me immediately is the timing chart: in February 2026, Claude Code shipped Agent Teams, Cursor shipped cloud agents, Codex CLI launched with Agents SDK, and Superset launched -- all within the same two-week window. The dossier presents this as a "why now" validation. I read it as a five-alarm fire. When every major platform simultaneously ships the exact capability your startup provides, you're not riding a wave -- you're standing in the path of one. Claude Code's Agent Teams already does native multi-agent orchestration without a separate tool. Cursor's cloud agents run on isolated VMs within their IDE. These aren't future threats -- they shipped the same month Superset launched. The company's entire value proposition is already a feature inside its dependencies.

Let me run my sane-person-insane-idea filter. The founders clear the "sane" bar convincingly -- Kiet Ho built Onlook to 23.4k GitHub stars and the #1 trending repo on GitHub, which demonstrates genuine open-source distribution instincts. Satya Patel brings infrastructure experience from Google, Amazon, and Meta. Three ex-YC CTOs who know how to ship developer tools. Credible team. But the idea? Building an orchestration layer for parallel AI coding agents is not insane -- it's the most obvious play in developer tools right now. Every VC who follows the space can see that multi-agent coding is happening. The "insane idea" premium is zero because the opportunity has been fully priced in by the market. This is a sane person with a sane idea, and that's the quadrant where I expect commoditized returns. Also notable: two of three co-founders left YC W25 companies to start this, and they appear to have connected through the YC network rather than through deep pre-existing relationships. I pay attention to that.

The structural moat question is where this falls apart for me. Superset is pure software orchestrating other people's software. Remove AI from the picture and this company has no reason to exist -- it fails my "cherry on top" test as cleanly as any company I've seen. The moat isn't just weak; it's actively degrading as the upstream platforms improve. The dossier itself notes that "the core concept of running multiple terminal sessions in isolated Git worktrees is technically reproducible" and that "any IDE or terminal emulator could add similar functionality." When you combine that with Cursor's $29.3 billion valuation and $2 billion-plus in ARR, the competitive calculus is brutal. Cursor can add every feature Superset offers as a checkbox in their settings panel. This is the exact category of software business I've been warning about since 2024 -- the moat dissolves faster than you can build it because AI makes the replication trivial.

The strongest bull case would go something like this: the AI coding agent market fragments into five or six serious players, and developers end up using three or four agents regularly, creating genuine demand for a vendor-neutral orchestration layer. Superset becomes the Kubernetes of agent orchestration -- the neutral middleware that no single agent provider has an incentive to build well because it supports competitors. The 6,500 GitHub stars in a short window show real developer pull. If Superset establishes itself as the default workflow before any single agent wins, switching costs could compound through configuration, team adoption, and ecosystem plugins. For this to work, though, you'd need to believe that cross-agent orchestration is a persistent, growing use case rather than a transitional one -- and that the agents won't converge on interoperability standards that make a standalone orchestrator unnecessary. I don't believe either of those things. The historical pattern is that developer tooling consolidates around platforms, not middleware layers. And the agents are getting better at self-coordination, which means the orchestration need shrinks as the technology matures.

The protocol-versus-application test seals it. Superset sits on top of Git worktrees, CLI agents, and IDEs -- all someone else's primitives. It doesn't create a new standard that other tools integrate with. It doesn't generate compounding network effects. It's a convenience layer. Compare this to Venmo, which created a new payment primitive that other applications built on, or Helium, which established a wireless protocol that became infrastructure. Superset is the inverse: it depends on every layer below it and controls none of them. The "agent-agnostic" positioning sounds like protocol-layer thinking, but it's not -- it's compatibility, not a standard. And with zero revenue, zero pricing page, and a free open-source model, there's no economic floor to cushion the narrative if adoption stalls. The GitHub stars are real traction, but Birchbox had real traction too. Traction without structural moat is the most expensive lesson in venture.
