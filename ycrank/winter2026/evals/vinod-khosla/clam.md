# Clam (formerly Baseframe) -- Vinod Khosla Evaluation

Here's what strikes me most about Clam's competitive landscape: every independent AI security startup of meaningful scale got acquired in 2025. Lakera, Protect AI, CalypsoAI, Prompt Security — all absorbed by Check Point, Palo Alto, F5, SentinelOne respectively, for a combined ~$1.2 billion. On the surface, this looks like a cleared field. An opening. But I've seen this pattern before, and what it actually tells me is that the cybersecurity incumbents have decided this category belongs to them. When Palo Alto Networks pays $500 million for Protect AI and plugs it into a platform already deployed at 80,000 enterprises, the distribution advantage is overwhelming. Clam isn't competing against startups anymore — it's competing against bundled features inside platforms that already own the enterprise security buyer relationship. That's a very different game than the one the pitch deck implies.

Let me start where I always start: if Clam succeeds completely, what changes? AI agents run more safely behind a semantic firewall. PII doesn't leak. Prompt injections get intercepted. Credentials stay out of the AI's context window. That's genuinely valuable — but it's security infrastructure for someone else's revolution. It doesn't displace an industry. It doesn't make an existing system economically obsolete. It makes an emerging system less dangerous. The difference matters enormously in my framework. When I backed Juniper, the consequence wasn't "Cisco routers work better" — it was "Cisco's entire architecture becomes irrelevant." When I backed Square, it wasn't "cash registers are safer" — it was "the traditional POS system disappears." Clam is building a better lock for a new kind of door. Important? Yes. Transformational? No. The market projections — $410 million today growing to $3.2 billion by 2033 — confirm this: it's a real market, not a civilization-scale disruption. I reserve my highest conviction for problems where success means the world operates fundamentally differently.

The bull case deserves serious engagement. If AI agents proliferate the way I believe they will — and I do believe autonomous agents will be ubiquitous within three years — then the network-level security layer becomes infrastructure, not a feature. The Cloudflare analogy is the best version of this argument: a proxy that sits between every agent and the outside world, accumulating data on attack patterns, building proprietary detection models, becoming the default security layer through developer-first adoption. The credential injection architecture — keeping secrets entirely out of the AI's context — is a genuinely smart design choice that addresses a real vulnerability no amount of prompt engineering can fix. And the founders' pivot speed during the batch, from hardware design validation to AI agent security, could signal exactly the kind of rapid learning I look for. Anshul's operational experience at HappyRobot, where he led AI evals and enterprise integrations for a company serving DHL and Schneider, means he's lived the pain of deploying AI agents in production. That's not theoretical understanding — he watched the security gaps emerge firsthand. If Clam can move fast enough to become the default SDK for AI agent deployment before the incumbents finish integrating their acquisitions, the 12-24 month window could be enough.

But the things that would make me write a check are absent. There's no contrarian signal here. Nobody is telling these founders they're crazy. The entire cybersecurity industry agrees that AI agents need security layers — that's why four companies got acquired for a combined billion-plus dollars. When everyone agrees the problem is real and the approach is sensible, I start worrying that we've de-risked the thesis to the point where the outcome is a competent business, not a category-defining one. The founders are smart Berkeley engineers with directly relevant experience building exactly the product the market expects. That's the profile other VCs optimize for. I optimize for outsiders attacking problems the consensus dismisses as impossible. The tight coupling to OpenClaw is another concern — "Run OpenClaw Securely in Minutes" is a positioning statement that trades breadth for adoption speed, and if OpenClaw builds native security features or loses market share to a competing framework, Clam's entire wedge narrows. The core scanning capabilities — PII detection, prompt injection detection, malicious code scanning — are individually addressable by open-source tools like LLM Guard and by any security team with resources. The bundled network-level approach is differentiated, but it's an architectural choice, not a moat.

The timing is real — AI agent deployment is accelerating exponentially, and I do track these trajectories carefully. But unlike my OpenAI investment in 2019, when the AI trajectory was clear but most VCs thought it was laughable, AI agent security in 2026 is consensus. The puck is already here. The rate of change in the underlying technology (agent capabilities) favors security tooling demand, but the rate of change in competitive response (incumbents integrating acquired products) may be faster than Clam can build. This is a market where the opportunity is obvious and the execution race is already underway — not the kind of asymmetric setup where I make my best bets.
