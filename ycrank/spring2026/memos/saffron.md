## The Idea

- **Problem:** Hiring teams cannot evaluate how candidate software engineers actually use AI agents (Claude Code, Cursor) to build, debug, and ship features, because legacy coding tests use sandboxed algorithmic puzzles rather than real codebases (trysaffron.ai, 2026).
- **Approach:** Candidates complete assessments on the company's actual codebase via native Claude Code integration; Saffron captures every keystroke, prompt, and diff, then runs 10+ independent AI scoring agents plus AI-powered debrief questions, with line-by-line human-vs-AI code attribution (trysaffron.ai feature list, 2026).
- **Differentiation:** Versus HackerRank/CodeSignal/Codility (puzzle-based, sandboxed environments per HackerRank.com comparison pages, 2025) Saffron evaluates work on the customer's real repo with full session replay; versus Karat (human-conducted live interviews-as-a-service per getlatka.com) Saffron is async with "zero interviewer hours required" (trysaffron.ai, 2026).
- **Business Model:** SaaS tiered: Basic $199/mo (5 assessments), Standard $499/mo (15 assessments, "Most popular"), Enterprise custom (unlimited); overage $49/assessment; each assessment includes $5 Claude Code budget (trysaffron.ai pricing page, 2026).
- **TAM/SAM:** No public TAM/SAM data found for this specific segment; nearest comparable ARR proxies in the technical assessment category: HackerRank $221M ARR (getlatka.com, 2024), CodeSignal $53.8M revenue (getlatka.com, 2024), Karat $73.6M ARR (getlatka.com, 2024).
- **GTM / Distribution:** [Inferred]: Founder-led outbound demo booking via Calendly (calendly.com/robert-trysaffron/demo on trysaffron.ai, 2026); no public content marketing, partnership, or channel disclosures found.

## Defensibility

- **Moat today:** [Inferred]: Early data moat from captured keystroke/prompt/diff traces used to train scoring agents — no public dataset size disclosed; native Claude Code integration cited on trysaffron.ai (2026) is a feature, not a contractual exclusivity.
- **Future moat:** [Inferred]: Proprietary labeled dataset of candidate AI-coding sessions scored against hire/no-hire outcomes could improve rubric accuracy; unproven now because no customer count, completed-assessment volume, or model performance benchmark is published (trysaffron.ai, 2026).
- **Market structure:** [Inferred]: Incumbents (HackerRank, CodeSignal, Codility) have built businesses around sandboxed proctored puzzle assessments and anti-AI cheating detection (per HackerRank.com 2025 product comparison pages) — re-architecting around "candidate uses AI freely on a real repo" partially cannibalizes their existing anti-cheating product lines.
- **Commoditization risk:** Established competitors HackerRank, CodeSignal, Codility, iMocha, Coderbyte are already shipping "AI IDE" and "AI scoring" features (HackerRank.com, 2025 comparison articles); Anthropic's Claude Code is an open consumer/dev tool, so the integration itself is replicable.

- **Incumbent feature replication:** HackerRank, CodeSignal, Codility, and Coderbyte are publicly shipping "AI IDE" and "AI scoring" modules in 2025 (hackerrank.com comparison articles, 2025); they hold distribution into existing enterprise ATS integrations and could ship a real-codebase mode using the same Claude Code APIs available to Saffron. No mitigation disclosed.
- **Dependency on Anthropic / Claude Code:** Saffron's product is built around "Native Claude Code integration" and bundles a "$5 Claude Code budget per assessment" (trysaffron.ai pricing, 2026); pricing, API access changes, or Anthropic shipping a first-party hiring evaluation feature would directly hit unit economics and product surface area.
- **Candidate-experience / IP risk on real codebases:** The product requires candidates to work on the company's actual codebase (trysaffron.ai, 2026); [Inferred]: enterprise security/legal review of exposing production repos to candidates is a non-trivial sales-cycle barrier and could narrow SAM to companies willing to grant repo access pre-hire.
