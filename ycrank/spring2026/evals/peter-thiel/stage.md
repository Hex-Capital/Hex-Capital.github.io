# Stage -- Peter Thiel Evaluation

The competitive landscape section of this dossier is its own indictment. CodeRabbit at $88 million raised and $15 million ARR. Greptile at $29 million. Graphite absorbed by Cursor for north of $290 million. Haystack from YC one batch earlier pursuing the identical thesis with a different visual metaphor. Cubic from YC two batches ago. Anthropic launching its own code review product in March 2026. When I count five funded competitors attacking the same problem on a single page, I am not looking at a market opportunity — I am looking at a battlefield where the combatants are about to destroy each other's margins. All failed companies are the same: they failed to escape competition. Stage is entering a war zone and calling it a greenfield.

The canonical question applies: what important truth has Stage discovered that the market disputes? The answer appears to be that code review diffs need narrative structure — "chapters" that organize changes into logical sequences rather than flat file-ordered output. But this is not a secret. It is a complaint. Every engineer who has reviewed a 500-line pull request knows that GitHub's diff UI is inadequate. The inadequacy is consensus. Haystack knows it. Graphite knew it. Every developer tools investor in San Francisco knows it. A secret must be something that makes knowledgeable people uncomfortable or dismissive — not something they nod along to while funding five competitors in the same direction. Stage's specific design choice — linear chapters versus Haystack's infinite canvas versus Graphite's stacked PRs — is an aesthetic preference masquerading as a contrarian insight. A UX metaphor is not a monopoly.

The deeper structural problem is that code review is a feature, not a platform. Graphite's trajectory proves this: the endpoint for standalone review workflow tools is absorption into a larger development platform. Cursor acquired Graphite not because Graphite had built an independent monopoly, but because stacked-PR workflow was a useful feature to embed. This is the ceiling that Stage faces regardless of execution quality. The best realistic outcome is a mid-nine-figure acquisition by GitHub, GitLab, Cursor, or whatever IDE platform consolidates next. That is a fine outcome for the founders. It is structurally irrelevant to the power law. A $200-300 million acquisition does not return a fund. I need investments that could become the last mover in a category worth tens of billions — and "organized diffs" is not that category.

The bull case deserves honest engagement. The timing argument has real force: AI-generated code is creating unprecedented volumes of unfamiliar changes that human reviewers must evaluate, and the tools have not caught up. Dean Stratakos built an in-house coding agent at Five Rings, which means he has lived inside the exact problem — watching AI-generated code overwhelm human review capacity. The co-founders share Stanford CS and overlapping tenure at a high-intensity quant firm, which is the kind of shared prehistory that produces aligned founding teams. If Stage's "chapters" approach proves to be the natural UX paradigm for AI-era code review — the way pull requests themselves became the paradigm for collaborative development — there is a world where they define the category. But that world requires Stage to be the last mover in code review UX, and nothing in the dossier suggests they possess the technological discontinuity or distribution advantage to achieve that position. The "chapters" concept is replicable by any well-resourced competitor in weeks, not years. GitHub could ship this as a native feature in a single release cycle.

The founders are competent — Stanford CS, quant trading at Five Rings, relevant AI experience — but competence in a competitive market is the definition of what I avoid. The dossier reveals no distribution strategy beyond the generic "bottom-up developer adoption" that every dev tools company claims. No traction, no users, no waitlist, no LOIs, no evidence of organic pull. This is a pure software layer with no physical-world barrier, no proprietary dataset, no network effect, and no government customer pathway. It is the kind of company that a lean-startup advisor would love: a real pain point, a reasonable first product, a team that can iterate. But iteration in a crowded market is not a plan — it is a prayer to the gods of mimetic competition. I invest in conspiracies to change the world, not in better diff viewers.
