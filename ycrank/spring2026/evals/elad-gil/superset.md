# Superset -- Elad Gil Evaluation

The first thing I notice is that Anthropic has already shipped native multi-agent orchestration for Claude Code. That's not a hypothetical competitive threat -- it's a current reality. When the platform your product depends on is actively building the feature you're selling as a standalone product, the structural dynamics are working against you. This is the core problem with Superset: it's an orchestration layer sitting on top of platforms that have every incentive and capability to absorb that layer themselves. Cursor, VS Code via Copilot, JetBrains -- any of these could add multi-agent management to their existing installed base of millions of developers. The dossier itself acknowledges the barrier is "UX polish and ecosystem integration, not deep technical complexity." When the moat is UX polish, you don't have a moat.

The market structural moment is real but consensus. CLI-based AI coding agents emerged in 2025, developers want to run multiple agents in parallel, and the orchestration workflow is genuinely painful today. I don't dispute any of that. But this isn't a non-obvious market -- it's squarely in the hottest investment category in technology. The TAM slides cite $10 billion growing to $91 billion. Every investor on Sand Hill Road is looking at AI developer tools. Compare this to PagerDuty, where operations alerting seemed so boring that most VCs dismissed it, or Gusto, where payroll seemed too mundane to build a venture-scale business. Those markets triggered dismissive reactions. AI coding tools triggers the opposite -- feeding frenzies. When I'm evaluating markets, I want to find the gap between perception and reality. Here, perception and reality are aligned: everyone knows AI coding tools are big, and everyone is funding them. The asymmetric returns live in markets other investors undersize or dismiss, not in the consensus frontier.

The product-to-distribution trajectory concerns me. Stripe's initial API wasn't just a payment tool -- it was a developer integration point that created a relationship Stripe could use to distribute Atlas, Billing, Radar, and Treasury. What does Superset's orchestration layer distribute? The company doesn't own the AI agents, doesn't proxy API calls (users supply their own keys), and doesn't control the value-creating layer. It's a management UI on top of other companies' products. Enterprise features like team collaboration and admin controls are reasonable upsells, but they don't transform a management layer into a distribution channel. The agent-agnostic positioning, which sounds like a strength, actually limits the depth of integration -- if you support every CLI agent equally, you can't build deep workflow abstractions on any of them. Figma didn't become a generational company by being a thin layer on top of Sketch and Illustrator. It replaced them with a fundamentally new collaborative architecture.

The strongest bull case: agent fragmentation is real. Developers may settle on Claude Code for refactoring, Codex for greenfield code, Gemini for documentation -- and no single agent vendor has incentive to orchestrate competitors' tools. If the future is multi-vendor AI coding (the way enterprises use multiple cloud providers), an agent-agnostic orchestrator has structural value that no incumbent will build. The 8,000 GitHub stars in five months validates real developer demand for this workflow, and the structural incentive misalignment -- Cursor won't build great support for Claude Code because it undermines their proprietary agent -- creates a genuine gap. If this team can establish category leadership before platforms consolidate, the orchestration layer could evolve into the developer's control plane for AI-assisted coding. That's a real product category, not a feature. For this to work, though, you need two things simultaneously: CLI agent proliferation continues rather than one agent winning (typical dev tool markets consolidate to 1-2 leaders), AND platform vendors don't absorb orchestration into their own products. That's two miracles, and compounding two low-probability events is exactly the pattern I avoid.

The founders are the strongest signal here. Three ex-YC CTOs who can clearly ship: Kiet Ho built Onlook to 25,000 GitHub stars, demonstrating rare ability to create open-source developer tool adoption. Getting to 8,000 stars in five months with a three-person team shows genuine execution velocity. The Product Hunt #1 daily launch and testimonials from builders at Microsoft, Netflix, and Google indicate real developer resonance. Satya's engineering tenure at Google, Amazon, and Facebook adds enterprise workflow understanding. These are builders who attract other builders -- that's a meaningful signal. But strong founders in a structurally challenged market is a configuration I've seen before, and the market usually wins. Many great teams get taken out by a terrible market structure.

The technology cycle timing is correct for right now -- CLI agents are proliferating, the pain is acute, and no orchestration standard has won. But the window feels narrow. The speed at which Anthropic shipped native multi-agent features for Claude Code suggests the major agent vendors view orchestration as core product, not ecosystem opportunity. Building a company where your "why now" window might be 12-18 months before platforms absorb your functionality is a timing bet I'm not comfortable with. When I backed Harvey, the timing advantage was structural: legal workflow expertise would compound over years even as model capabilities improved. Here, the timing advantage could evaporate in a single platform update.
