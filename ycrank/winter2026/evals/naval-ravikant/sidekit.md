# SideKit -- Naval Ravikant Evaluation

The competitive landscape here is what stops me before anything else. SideKit is bundling three commodity capabilities -- analytics, feature flags, version gating -- in a market where Firebase is free and embedded in 72% of iOS apps and essentially all Android apps, PostHog has $180M+ in funding and already bundles analytics with feature flags as open source, and LaunchDarkly owns enterprise feature management with $330M behind it. Bundling well-understood components into a lighter package is a feature, not a company. Any one of these incumbents could add version gating in a sprint. The structural question isn't whether SideKit can build a nice SDK -- it's whether bundling creates enough switching cost to survive Firebase being free. History says no. When your moat is "we're simpler," the incumbents eventually get simpler too.

The specific knowledge question is where this falls short of my threshold. Ashish Selvaraj built Not Evil Sudoku, got 50K installs, and then built a private analytics service for it because existing tools didn't fit. That's genuine pain. That's a real origin story. But it's also an experience shared by thousands of indie iOS developers who ship apps and discover that Firebase is bloated and Mixpanel is expensive. Building one consumer app and needing better analytics is common developer frustration, not the kind of deep, non-transferable domain knowledge that takes competitors years to accumulate. Compare this to what I look for: Amjad Masad had been building coding environments for years before Replit, understanding something fundamental about the gap between wanting to code and being able to code. The SideKit founders are smart Waterloo CS graduates with strong internship pedigrees -- Databricks, Microsoft, and claimed but unverified Jane Street and Citadel experience for the co-founder. That's engineering talent, not domain-specific insight that can't be taught.

The one element that genuinely interests me is the self-healing AI feature -- agents that process user feedback and ship fixes as pull requests. If this works, it's code writing code, which is real leverage. It's the only part of SideKit that couldn't be replicated by a competent team in a few weeks. But there's no evidence it works. No customer testimonials, no case studies, no public demonstration. And the technical claim is ambitious -- reliably generating correct code fixes from unstructured user feedback is a hard problem that even well-resourced teams haven't solved generally. If the AI fixes introduce regressions, trust in the entire platform collapses. This is a feature that's either a genuine breakthrough or a marketing slide, and at this stage I have no way to distinguish between the two.

The bull case deserves honest engagement. The AI coding explosion -- Cursor, Copilot, Claude generating code -- is producing a new class of builder who can create apps but has no idea how to manage production. SideKit positions as the bridge between "I built an app" and "my app is in production." If the AI coding wave produces 10x more indie apps over the next three years, the market for simple production infrastructure genuinely expands. The 244KB SDK footprint is a real engineering discipline signal -- lightweight matters in mobile, and it shows the founders care about the developer experience at a granular level. The free version-gating tier is a smart wedge: it solves an immediate pain point for any developer who's shipped a buggy version and needed to force an update. If they nail the onboarding funnel from free version gating to paid analytics to self-healing AI, there's a plausible path to $10M ARR serving the long tail of mobile developers that Firebase and LaunchDarkly don't optimize for. But this requires them to win against PostHog (which is open-source, well-funded, and building the same bundled thesis for web), survive Firebase's gravitational pull, ship Android support, and prove the self-healing feature actually works. That's four independent conditions that all need to be true simultaneously.

The unverified credential claim nags at me. The website says "engineers from Databricks, Jane Street, Citadel, Microsoft." Ashish's Databricks and Microsoft internships are confirmed. But no public profile for Ethan Pronev could be found -- no LinkedIn, no GitHub, no confirmable Jane Street or Citadel connection. This could be innocuous -- maybe he's private online. But claiming prestigious firm names on your marketing page when they can't be independently verified is exactly the kind of small behavioral tell I watch for. It's not disqualifying, but it doesn't build trust either.

The leverage architecture is standard SaaS -- write software, sell subscriptions, scale with code. Nothing wrong with that as a baseline. But there are no network effects here. One developer using SideKit doesn't make it better for another developer. The self-healing AI could theoretically create a data flywheel -- more feedback across more apps improves the fix quality -- but that's a hypothesis, not a mechanism. Without a compounding dynamic, this is a linear business competing on product quality against teams with 100x more resources. I invest in companies where the leverage ratio gets better over time, where each additional user creates value for existing users. SideKit at $99/month with 5M signals is a useful tool. It's not a platform that compounds.
