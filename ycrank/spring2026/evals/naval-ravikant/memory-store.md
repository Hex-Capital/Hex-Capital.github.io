# Memory Store -- Naval Ravikant Evaluation

Memory Store is what happens when a four-person team pulls a literal `src/memory-store` directory out of their prior open-source agent platform and ships it as a standalone product ? extreme team-to-value leverage paired with weak access democratization. Diwank Tomer and Ishita Jindal spent three years (2022?present) building Julep AI to 6.6k GitHub stars and a reported $400k ARR, and Memory Store is the extraction of the one component they discovered users actually wanted. That high/low pairing tells me what kind of bet this is: a tools-for-builders SaaS with real founder-specific knowledge but no structural gatekeeper to remove ? closer to Notion's profile than to AngelList's.

On team-to-value leverage, the ladder runs from Midjourney (163 people, $500M revenue) and Cursor (<50, $500M ARR) at the top, to a typical YC pre-seed team in the middle, to a services company hiring 50 ops staff at the bottom. Memory Store sits closer to the top rung than to the middle: four people, technical builders shipping nine MCP integrations (Claude, ChatGPT, Cursor, Slack, Raycast, Linear, Granola, Fathom, OpenCode), founder-led onboarding through Ishita's Cal.com link, no operations layer. Diwank shipped Julep at 6.6k stars with the same lean structure ? this isn't a team that scales by hiring, it's a team that scales by writing code other agents call.

On access democratization Memory Store is structurally weak. AngelList sits at the top of this ladder ? it took startup investing away from a closed Sand Hill insider class. Notion sits at the middle ? a better tool for an already-served knowledge-worker market. An incremental SaaS in a crowded category sits at the bottom. Memory Store does not remove a gatekeeper; it stitches together context across tools that already serve the same user. The dossier offers no evidence that a previously-restricted population now gets access to anything ? "memory across AI agents" is a productivity gain, not a structural unlock. This is the dimension where Memory Store most clearly fails my unstated portfolio pattern.

Specific knowledge density is the dimension that almost saves the deal. The HIGH rung is Perplexity ? Srinivas's years inside Google Brain and OpenAI productized into AI search. The LOW rung is the MBA AI-wrapper team I always pass on. Memory Store sits above the middle: Diwank and Ishita didn't identify "AI memory" through market research, they ran into the problem inside Julep and shipped a memory-store component before extracting it. That is specific knowledge revealing itself through revealed behavior, not a pitch. It is not Perplexity-grade ? memory primitives are more replicable than the systems Srinivas built ? but it clears my "could be replaced by another competent operator" threshold.

Permissionless leverage looks like Notion in the middle of the ladder, Twitter at the top, and Poshmark's marketplace-ops team at the bottom. Memory Store is code leverage, but rented code leverage ? Anthropic owns the MCP standard, and Slack, Fathom, Granola, and Linear own the APIs Memory Store ingests. That's the AI-wrapper risk in disguise. Anthropic shipping native long-term memory in Claude (which it has already begun in 2025) is the same threat OpenAI shipping native summarization poses to my pass example. Memory Store sits below Notion on this rung.

Principal authenticity is real but ambiguous. Sahil Lavingia at Gumroad sits at the top ? kept coding alone after the failed Series A. The bottom is the founder who pivoted into a fundable space after an unrelated failure. Memory Store sits above the middle: Diwank and Ishita are spouses with three years co-building Julep, and Memory Store is a focusing of that work, not a swerve into something fundable. The asterisk is that the dossier does not say whether Julep is wound down or running in parallel ? splitting principal attention is the precise risk Naval's framework flags.

Memory Store reaches $1?3M ARR within 18 months on bottom-up MCP adoption inside Claude- and Cursor-heavy SF teams, then gets acquired by Anthropic, Notion, or Linear at a $40?80M outcome within 36 months as native memory features commoditize the standalone wedge. The mechanism is exactly the team-to-value-high / democratization-low signature: four builders move fast enough to win the integrations layer, but no structural moat prevents the platform owners from absorbing the category once the user behavior is proven.

The bear case stings. Mem0 has raised $24M and reports 50,000+ developers; Letta has 13k+ stars; Zep, Supermemory, Cognee, and LangMem all ship overlapping primitives. My own red flag reads almost verbatim: "a slightly better version of an existing product in a crowded market." Memory Store's defense is the team-shared/MCP packaging rather than the developer SDK ? a real positioning difference, since Mem0 sells to developers and Memory Store sells to teams using AI agents. That is enough wedge for an angel-sized convex bet but not for conviction. The condition that flips me to a strong yes: Memory Store hitting $500k ARR on team plans within nine months without paid acquisition ? organic team-level pull would prove the MCP positioning is structurally different from the developer-SDK incumbents, not just a marketing layer over the same primitive.
