# Superset -- Sam Lessin Evaluation

Here's the thing that should terrify anyone writing a check into Superset: Anthropic has already shipped native multi-agent orchestration for Claude Code. Already. It's in the docs. The upstream platform provider looked at the exact workflow Superset is productizing and said "yeah, we should just build that." This is not a hypothetical risk I'm pattern-matching from other categories -- this is happening in real time, right now, while Superset is in YC. Every single AI agent provider has the same incentive. OpenAI, Google, Cursor -- they all want you inside their ecosystem running their agents, and "let me manage your parallel sessions" is a Tuesday afternoon feature sprint for any of these teams. This is the canonical case I keep banging the table about: when I say "every time you get pitched Adobe for AI, think about whether this is just perfect for Adobe," Superset is the literal textbook example. Replace Adobe with Anthropic. Replace it with Cursor at their $29.3B valuation and $1.2B ARR. Replace it with VS Code and its hundred million users. The orchestration of CLI agents into parallel git worktrees is a feature, not a company.

And the mispricing question kills this before we even get to the product. AI developer tools is the single hottest consensus category in venture capital right now. Every smart investor I talk to is trying to get into the next Cursor. The pricing already reflects the full upside of the category. There is no narrative mispricing here -- there's narrative premium. When I seeded Solana, literally every crypto investor was telling me Ethereum had won and I was wasting money. That's a pricing dislocation. Superset is the opposite: it's a company in a category where every VC is salivating, which means whatever valuation they're raising at already bakes in the optimistic scenario. The risk isn't in the price -- the hope is in the price.

The primitive test is where I'd want to get excited, and I can't. Multi-agent orchestration is not a new behavior -- it's a convenience layer over an existing workflow. Developers already run multiple agents. They already use git worktrees. They already have tmux. Superset makes it 3x nicer, and that's genuinely valuable, but "nicer" doesn't create a category. Venmo created peer-to-peer mobile payments as a behavior that literally didn't exist before -- people started saying "Venmo me." Nobody is going to say "Superset me." Nobody is going to describe a new category of human activity by reference to this product. It's a better way to manage something developers already do, which means it competes on execution in an existing behavioral pattern rather than inventing a new one.

Now, the bull case I have to honestly engage with: maybe the world fragments into dozens of competing AI coding agents, and the orchestration layer becomes the new IDE -- the thing developers actually live in while agents come and go underneath. The "Docker for AI agents" analogy. Kiet Ho built Onlook to 25k GitHub stars, which is legitimately impressive open-source distribution. 8k stars in five months for Superset shows the team can ship and attract developers. If you believe the agent landscape stays permanently fragmented and no upstream provider successfully bundles orchestration, this is a real wedge into a potentially large workflow category. That's the world where this works. But I don't believe that world persists. Agent providers have overwhelming incentives to own the orchestration layer because it's where user lock-in lives. And the co-founders, while technically sharp, met through overlapping YC batches -- not through the kind of deep multi-cycle relationship that gives me conviction. Three CTOs who each left their previous YC startup to start something together is... fine. It's not the pattern that produces my best returns, which come from people I've watched grind through multiple cycles over years.

The structural durability problem is not subtle -- it's the loudest signal in the entire dossier. Pure software wrapper around third-party APIs and CLI tools, no physical-world operations, no protocol-level network effects, no proprietary data moat that compounds over time. The 8k GitHub stars are real community traction, but community alone doesn't survive when Anthropic, OpenAI, and Cursor are all building the same surface area natively. Claude Squad already does the basic version for free under AGPL. The Elastic License 2.0 choice is pragmatic but limits the true open-source community energy that could be a differentiator. I keep coming back to the same conclusion: software companies that had moats are discovering they aren't moats anymore, and Superset is trying to build a moat that never existed in the first place.

Capital-wise, three people in SF with no disclosed revenue is standard YC pre-seed. Nothing alarming, nothing exciting. The lean team is fine. But "fine" doesn't move the needle when every other dimension is flashing amber or red. I'm passing. This is a well-executed product in exactly the wrong category for my framework -- consensus-hot, application-layer, zero structural durability against upstream commoditization, and no new primitive being created. The founders are competent, the product is real, and none of that matters when Adobe is already building your feature.
