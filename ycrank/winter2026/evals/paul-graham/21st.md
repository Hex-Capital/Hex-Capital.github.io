# 21st -- Paul Graham Evaluation

Nine products in ten months. That number jumped off the page at me, and not in a good way. When I see founders shipping that fast across that many surfaces, I don't see relentless resourcefulness -- I see people who haven't found the problem they can't stop working on. Serafim went from a cross-chain crypto protocol to a memecoin launchpad to an AI video editor to a component registry to agent infrastructure. Each one of those is a real category. None of them appear to have held his attention for more than a few months. The best founders I've funded are the opposite: they're obsessed with one problem, often for years before they start a company. Drew Houston couldn't stop thinking about file sync. The Collisons couldn't stop thinking about payment APIs. These founders can't stop thinking about... launching things.

The origin of the idea is where my concern crystallizes. The component registry -- the original product -- passes the organic test. Developers building shadcn/ui components for other developers is the kind of thing that grows naturally from someone's own workflow. You're building UIs, you notice the same patterns recurring, you extract and share them. That's how good developer tools start. But somewhere around mid-2025, the company pivoted to "infrastructure and UI building blocks for the agentic internet," and that's a phrase I could have generated by feeding TechCrunch headlines into GPT. The expansion to Agents SDK, Magic MCP, and 1Code happened precisely when every developer tools company started chasing the AI agent wave. When I ask "how did they find this problem?" the honest answer appears to be: they watched what was trending. That's the failure mode I've written about extensively -- smart people generating ideas from market analysis rather than personal pain.

The schlep test is even more revealing. UI components are the opposite of a schlep -- they're fun to build, easy to showcase on Product Hunt, and inherently commoditizable. Any competent React developer can build shadcn-based components. The agent infrastructure layer could theoretically involve real schlep (sandboxing, security, observability at scale), but 21st hasn't demonstrated depth there yet -- the Agents SDK is in beta with all features free, which means they haven't even tested whether anyone will pay for it. Meanwhile, Vercel just launched AI Elements -- open-source React components built on shadcn/ui specifically for AI interfaces -- with 20M+ monthly downloads on their AI SDK providing distribution that a three-person startup cannot match. This is exactly the competitive dynamic I worry about: the hard part isn't building the components, it's building the ecosystem, and Vercel already has the ecosystem.

The bull case deserves genuine consideration. These founders clearly know how to build and ship -- Sergey was a senior engineer at Deel, Serafim has two prior acquisitions, and the combined 14K+ GitHub stars and 200K MAU on the component registry represent real developer attention earned by a tiny team. The MCP server positioning is early-mover in a nascent protocol standard, and if MCP becomes the universal standard for AI IDE integration, being the most-referenced MCP server for UI generation is a valuable wedge. There's a version of this story where the component registry is the Trojan horse -- developers discover 21st for the free components, then adopt the agent SDK when they need to build production AI applications, and 21st captures the full stack. That bottom-up conversion path is how many great developer tools companies were built. If the founders stopped launching new products and went deep on one -- say, making 1Code the definitive orchestration layer for coding agents -- the execution velocity they've demonstrated could be formidable in a focused direction.

But I keep coming back to the structural problems. Four distinct products with a three-person team is fragmentation, not focus. The $20/month Pro tier on the component registry generates maybe a few thousand dollars monthly -- nowhere near ramen profitable, let alone default alive. The 1.4M developer number almost certainly aggregates users across all nine products launched over ten months, which makes it a vanity metric for evaluating the current agent infrastructure bet. And the competitive landscape is brutal: v0.dev has 3.5M users and ~$42M ARR backed by Vercel's $863M in funding. CopilotKit claims 10%+ of the Fortune 500. These aren't scrappy competitors -- they're well-funded incumbents moving in the same direction.

The founders are talented builders. That's not in question. What's in question is whether they've found their Stripe -- the problem that's been hiding in plain sight because the schlep scares everyone away. Right now, "agent infrastructure" isn't hiding in plain sight; it's the most visible category in developer tools. Everyone sees it. Everyone's building it. The problem 21st is solving isn't one that others unconsciously avoid -- it's one that dozens of funded competitors are actively pursuing. That's precisely the dynamic I try to avoid as an investor.
