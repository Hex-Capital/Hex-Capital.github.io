# Stage -- Sam Lessin Evaluation

$334.5 million. That's the combined funding behind CodeRabbit, Graphite, Greptile, and Qodo — the four companies already fighting over AI-era code review. Stage is walking into this with two people and a UX concept called "chapters." My first question with any deal is "what if they're right?" — but here the more honest question is "even if they're right, does it matter?" Because Stage isn't operating in a space the market has mispriced. Stage is operating in one of the most consensus-validated categories in developer tools circa 2026. Anthropic launched Claude Code Review in March 2026. Cursor acquired Graphite for $290 million in December 2025. Every sophisticated investor I know has a code review bet already placed. When every other investor at demo day is excited about your category, the upside is already in the price. There's no asymmetry left to exploit.

Stage's core innovation — breaking diffs into "structured chapters" rather than file-by-file views — is a presentation-layer rethink, not a new primitive. Compare this to Venmo, which created a behavior that literally didn't exist: sending money to your friend from your phone by typing their name. People started using "Venmo" as a verb because the behavior was genuinely new. Stage reorganizes how you look at something you were already looking at. Nobody's going to say "Stage me that PR." Code review exists. Diff viewers exist. Stage makes the diff viewer more narrative. That's a feature improvement in an established workflow, and my framework is pretty clear: "a better version of an existing product rather than a new category" is a pass signal. The "chapters" concept doesn't birth a category — it refines one.

Charles Pan (Stanford CS '22, Five Rings, early engineer at Yuzu Health) and Dean Stratakos (Stanford MS/BS CS, Five Rings AI lead, three Apple internships) are clearly sharp engineers. Stratakos building an internal coding agent at Five Rings gives him direct domain experience with the AI-generated code problem. Their co-founder relationship — overlapping at Stanford and Five Rings — is real, not synthetic, which I care about. But here's the thing: two quant firm engineers building a developer tool is the most legible, predictable founder path I can imagine. There's nothing insane about it. The idea is sensible. The founders are sensible. Sensible plus sensible is boring. When I backed Teamshares — employee-owned small business rollups as a venture bet — people thought the idea was unhinged. That's the ratio I look for. Stage doesn't have it.

Stage fails the durability test harder than almost anything I've seen this batch. The dossier itself concedes that GitHub's "structural barrier" to adding a chapters view "is weak" — they could ship it as an optional mode without breaking backwards compatibility for their 100M+ developers. CodeRabbit has $15M ARR and 8,000 paying customers; they could add narrative structuring as a feature next quarter. The "chapters" concept is explicitly described as "a UX innovation rather than a deep technical moat, making it reproducible by any competitor with front-end engineering resources." Stage is a pure software tool with no physical operations, no protocol-level lock-in, no network effects that survive a competitor cloning the interface. This is exactly the pattern I warned about on CNBC — software moats are dissolving. Stage doesn't even have the moat to dissolve.

The strongest bull case I can construct: AI-generated code is a genuine paradigm shift that 5-10x's the volume of changes per PR, and the comprehension bottleneck becomes the critical constraint. Stage is uniquely positioned as the only company focused purely on human comprehension rather than layering more AI commentary on top. If "chapters" becomes the standard mental model for consuming code changes — the way Figma's multiplayer canvas became the standard for design — Stage could define a new workflow paradigm that platforms eventually adopt but can't easily displace. The Cursor/Graphite acquisition at $290M proves code review startups can reach strategic exit velocity. And maybe GitHub's institutional inertia is stronger than the dossier suggests — their diff view has been essentially unchanged for a decade. But this bull case collapses on two facts: the "chapters" concept is reproducible with front-end engineering effort (no deep moat), and four competitors with $334.5M in combined funding are already shipping structured review features in 2025-2026. Stage would need to become the de facto comprehension layer faster than incumbents can copy it, with two people and YC-scale capital, against companies with 50-100x their resources. That math doesn't work. The condition that would flip me: if Stage showed evidence of a proprietary technical approach to chapter generation that competitors couldn't easily replicate — some deep code analysis primitive, not just a UI reorganization — and had demonstrable user retention data showing teams can't go back to raw diffs after using chapters, I'd reconsider. A UX innovation that creates genuine behavioral lock-in is different from a UX innovation that's just nicer. But nothing in this dossier suggests the former.

The two dimensions that lock this verdict are Consensus Mispricing and Structural Durability. Stage enters the most consensus-hot category in developer tools — the exact opposite of the mispriced contrarian bets I hunt for — and its core innovation is a presentation-layer feature that any platform or well-funded competitor can replicate. Pan and Stratakos are competent engineers solving a real problem, but the market has already priced this opportunity, and Stage brings no structural defense to the fight. Pass.
